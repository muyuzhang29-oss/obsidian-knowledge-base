# SerDes 常用术语解析

## 概述

本文档解释车载高速媒体流 SerDes 芯片中常见的专业术语，结合项目实践说明每个模块的功能和在验证中的关注点。

---

## 1. TDR — Test Data Register (测试数据寄存器)

### 全称与所属标准
- **全称：** Test Data Register
- **标准：** IEEE 1149.1 (JTAG)，IEEE 1687 (IJTAG)

### 在 SerDes 中的角色

TDR 是 JTAG 架构中串接在 TDI 与 TDO 之间的移位寄存器。在 SerDes 芯片中，TDR 主要用于：

| 用途 | 说明 |
|------|------|
| **PHY 边界扫描** | 通过 Boundary-Scan Register (BSR) 测试 SerDes 高速引脚（如 RX±, TX±）的焊接质量和 PCB 互联 |
| **内部扫描链** | 将 SerDes 数字核心（PAL、CDR 状态机、FIFO）的 DFT 扫描链作为用户 TDR 接入 JTAG，用于 ATE 测试 |
| **BIST 触发与结果回读** | 通过 IJTAG 网络访问各 IP 的 BIST 控制器（如 PAL_BIST、PLL_BIST），触发自检并读出签名 |
| **调试接口** | 在回片调试阶段，通过 JTAG TDR 访问内部寄存器和观测信号，无需功能接口正常工作 |

### 验证要点
- BSDL 文件中的 TDR 长度、BSR 单元位置必须与 RTL 一一对应
- SerDes 多通道的 TDR 链在 SoC 层级的拼接顺序（lane0→lane1→...→laneN）
- IJTAG SIB (Segment Insertion Bit) 的层级使能/旁路逻辑
- TAP 控制器 16 状态机的所有合法/非法转移

## 2. DPLL — Digital Phase-Locked Loop (数字锁相环)

### 全称
- **全称：** Digital PLL / All-Digital PLL (ADPLL)
- **相对于：** HPLL (模拟 PLL)

### 在 SerDes 中的角色

车载 SerDes 接收端（RX）通常使用 **DPLL 实现 CDR（Clock Data Recovery）**：

```
串行数据 (RX±) ──► 前端均衡器 ──► 边沿/数据采样器 ──► DPLL (CDR) ──► 恢复时钟 + 重定时数据
                              └── Bang-Bang PD ──► 数字环路滤波 ──► DCO ──► 恢复时钟
```

| 子模块 | 功能 |
|--------|------|
| **Bang-Bang 鉴相器 (BBPD)** | 比较数据边沿与恢复时钟的位置，输出 early/late 信号 |
| **数字环路滤波器** | PI 控制器，带宽可编程（适应不同串行速率），支持动态切换 |
| **DCO (Digitally Controlled Oscillator)** | 数字控制振荡器，产生多相位时钟（通常 8-16 相），用于数据过采样 |
| **频率锁定辅助 (FLL)** | 上电时先通过频率检测将 DCO 拉到目标频率附近，再切换到相位锁定 |

### 为什么用 DPLL 而不是模拟 PLL？

| 对比项 | DPLL (CDR) | 模拟 PLL (HPLL) |
|--------|------------|-----------------|
| **工艺可移植性** | RTL 综合，随工艺缩放 | 需重新模拟设计 |
| **带宽可配置** | 数字系数，PVT 不变 | RC 常数随 PVT 变化 |
| **多速率支持** | 系数表切换，灵活 | VCO 调谐范围受限 |
| **相位精度** | 受 TDC/DCO 分辨率限制 | 连续模拟，精度更高 |

> 车载 SerDes 通常：**HPLL 发 TX 时钟**（需要极低抖动），**DPLL 收 RX CDR**（需要灵活性和可移植性）。

### 验证要点
- TDC/DCO 的单调性验证（控制字每 +1，频率/相位必须单调变化）
- CDR 锁定时间：从无输入切换到有效数据，锁定时间需 < 协议要求（如 < 5 us）
- 长连 0/1 下的保持能力（抖动累积不超过 1 UI）
- 输入频偏容忍度（如 ±300 ppm）
- AMS 混合仿真：数字 CDR + 模拟前端/通道模型

## 3. HPLL — High-Performance PLL (高性能锁相环)

### 全称
- **全称：** High-Performance Phase-Locked Loop
- **类型：** 传统模拟电荷泵 PLL

### 在 SerDes 中的角色

车载 SerDes 发送端（TX）使用 **HPLL 产生高速串行时钟**：

```
参考时钟 (25MHz/50MHz) ──► PFD ──► 电荷泵 ──► 环路滤波器 ──► LC VCO ──► 串行时钟 (1.6Gbps/3.2Gbps/6Gbps...)
                               └───────────── ◄─ ÷N 分频器 ◄──────────────────┘
```

| 子模块 | 功能 |
|--------|------|
| **PFD (Phase-Frequency Detector)** | 检测参考时钟与反馈时钟的相位/频率差，输出 UP/DN 脉冲 |
| **电荷泵 (Charge Pump)** | 将 UP/DN 脉冲转换为充放电电流，驱动环路滤波器 |
| **环路滤波器 (RC)** | 二阶/三阶无源 RC 滤波器，设定 PLL 带宽（通常参考频率的 1/10~1/100） |
| **LC VCO** | 电感电容振荡器，产生 GHz 级串行时钟，相位噪声决定 SerDes 误码率 |
| **÷N 分频器** | 将 VCO 输出分频后反馈给 PFD，N 值决定输出频率 = Fref × N |

### 在车载 SerDes 中的关键指标

| 指标 | 典型要求 | 影响 |
|------|---------|------|
| **RMS 抖动 (12kHz~20MHz)** | < 1 ps rms | 决定 SerDes 误码率 (BER) |
| **相位噪声 @1MHz** | < -130 dBc/Hz | 通道串扰容限 |
| **锁定时间** | < 100 us | 上电/唤醒时间 |
| **参考杂散 (Reference Spur)** | < -60 dBc | 对相邻通道的干扰 |
| **频偏容忍** | ±300 ppm | 适应不同数据源 |

### 验证要点
- VCO 调谐范围是否覆盖所有目标速率（如 1.6G/3.2G/6G，含 PVT 余量）
- 电源噪声注入测试：模拟板上电源纹波（如 100mVpp @ 1MHz），验证 PLL 输出抖动
- 锁定检测器的可靠性：双重判据（频率差 + 相位差），防止误锁/假锁
- AMS 混合仿真：用 Verilog-AMS/RNM 模型在 testbench 中验证 PLL 行为

## 4. EFUSE — Electronic Fuse (电子熔丝)

### 全称
- **全称：** Electronic Fuse / eFuse
- **类型：** 一次性可编程 (OTP) 非易失存储

### 在 SerDes 中的角色

车载 SerDes 芯片使用 eFuse 存储**不可更改的芯片特有信息**：

| 用途 | 说明 |
|------|------|
| **模拟修调 (Trim)** | 存储 HPLL VCO 粗调码、DPLL DCO 校准码、Bandgap 基准修调值、RX 均衡系数——在测试时写入，芯片每次上电自动加载 |
| **芯片标识** | 芯片 ID、生产批次、晶圆坐标，用于追溯 |
| **安全密钥** | 车载 SerDes 需要加密传输视频流，eFuse 存储根密钥 |
| **冗余修复** | SRAM/FIFO 的冗余行列替换地址 |
| **Speed Binning** | 标记芯片能跑的最高速率（如 3.2G / 6G），用于分级销售 |
| **配置熔丝** | 永久使能/禁用某些功能（如安全特性、调试接口） |

### 工作原理（硅化多晶硅熔丝）

```
编程序列: 芯片上电 → 升压 (2x~3x VDD) → 字线选通 → 大电流 (10~30mA, 1~10us)
                                                                    ↓
                                                  硅化物电迁移 + 热扩散 → 电阻从 ~100Ω → >1MΩ

读取: 正常 VDD → 小电流 → 灵敏放大器比较位线电压与参考
```

### 验证要点
- **Program-Verify-Read (PVR)** 测试模式：编程后读出确认，若阻值不够则重试
- **上电加载序列**：复位释放 → eFuse 读取 → 修调码分发到各模拟 IP → 各 IP 进入工作状态，整个过程需在指定时间内完成
- **安全锁定**：安全熔丝一旦编程，JTAG 调试接口永久关闭，需验证锁定不可逆
- BIST：eFuse IP 自带的 Built-In Self-Test，需在测试模式下验证所有位可编程和可读取

## 5. GPIO — General Purpose Input/Output (通用输入输出)

### 全称
- **全称：** General Purpose Input/Output

### 在 SerDes 中的角色

车载 SerDes 芯片的 GPIO 用于板级控制与状态交互：

| GPIO 功能 | 方向 | 说明 |
|-----------|------|------|
| **LOCK / LOS** | 输出 | SerDes 链路锁定指示 / 信号丢失告警 |
| **PWDN / PD** | 输入 | 芯片/通道休眠控制 |
| **MODE[2:0]** | 输入 | 上电时配置工作模式：速率、视频格式、加密使能等 |
| **INT / FAULT** | 输出 | 中断/故障指示（CRC 错误、BER 超限、温度告警） |
| **SLEW_CTL** | 输入/输出 | 输出摆率控制 |
| **GPIO_SPI_CS** | 输入 | 片选（可配置为专用 SPI 或 GPIO 模式） |
| **strap[4:0]** | 输入 | 上电采样管脚，决定芯片 I2C 地址/链路角色（主/从） |

### 典型寄存器映射（APB 总线）

```
基址 + 0x00: DATA_OUT      — 输出数据寄存器
基址 + 0x04: DATA_IN       — 输入数据寄存器（只读）
基址 + 0x08: DIRECTION     — 方向控制（0=输入, 1=输出）
基址 + 0x0C: PULL_EN       — 上下拉使能
基址 + 0x10: DRIVE_STR     — 驱动强度选择
基址 + 0x14: INT_EN        — 中断使能
基址 + 0x18: INT_STATUS    — 中断状态（写1清除）
基址 + 0x1C: MUX_SEL       — 功能复用选择
```

### 验证要点
- 所有 GPIO 在复位后必须处于已知安全状态（通常输入+上拉，中断关闭）
- 每个 GPIO 的输入/输出/中断/上下拉/驱动强度组合随机验证
- Strap 管脚在复位释放时采样，之后切换为 GPIO 功能
- 多电压域的 Level Shifter 验证

## 6. PAL_BIST — PAL 模块内建自测试

### 全称
- **全称：** PAL Block Built-In Self-Test
- **PAL：** 在 SerDes 上下文中指数据路径中的并行-串行/串行-并行转换与帧对齐模块

### 在 SerDes 中的角色

车载 SerDes 数据流中的 PAL 模块负责：

```
TX 方向:
  并行数据总线 (video data + control) ──► [PAL_TX] ──► 帧编码/加扰/并串转换 ──► HPLL ──► 串行输出

RX 方向:
  串行输入 ──► HPLL CDR ──► 串并转换/帧同步/解码 ──► [PAL_RX] ──► 并行数据总线
```

**PAL 包含的子模块：**

| 子模块 | 功能 |
|--------|------|
| **帧编码器/解码器** | 将并行数据封装为 SerDes 帧格式（含同步头、数据类型标识、CRC） |
| **加扰/解扰器** | 对数据进行加扰以减少 EMI 和保证 DC 平衡 |
| **串行器 (Serializer)** | 将并行数据（如 10bit/20bit）逐位移出给 HPLL |
| **解串器 (Deserializer)** | 将恢复的串行数据组装为并行字 |
| **FIFO** | 跨时钟域缓冲（PAL 时钟域 ↔ 系统总线时钟域） |
| **帧同步状态机** | 检测同步头，锁定帧边界 |

**PAL_BIST 的测试架构：**

```
                    ┌──────────────────┐
                    │  BIST 控制器      │
                    └────────┬─────────┘
                             │
         ┌───────────────────┼───────────────────┐
         ▼                   ▼                   ▼
   ┌──────────┐      ┌──────────────┐      ┌──────────┐
   │ PRBS/LFSR│─────►│  PAL 模块    │─────►│  MISR    │
   │ (TPG)    │      │  (被测电路)   │      │ (签名器) │
   └──────────┘      └──────────────┘      └──────────┘
                             │
                    内部扫描链 (DFT)
```

| BIST 组件 | 功能 |
|-----------|------|
| **TPG (Test Pattern Generator)** | LFSR 或 PRBS 发生器，产生伪随机数据/控制信号 |
| **PAL 模块** | 被测电路，工作于 BIST 模式（输入来自 TPG，输出送往 MISR） |
| **MISR (Multiple-Input Signature Register)** | 将 PAL 输出压缩为单一签名值 |
| **BIST 控制器** | 调度 BIST 序列：初始化 → 运行 → 比较签名 → 输出 pass/fail |
| **Golden Signature** | 预计算的标准签名，在 RTL 仿真中确定 |

### 验证要点
- **BIST 隔离**：BIST 模式下 PAL 必须与功能总线隔离，避免总线冲突
- **PRBS 序列正确性**：验证 LFSR 多项式生成了预期的伪随机序列
- **签名比对待故障注入**：翻转 PAL 内部一个 bit，验证 MISR 签名变化，BIST 报 fail
- **全速测试**：BIST 跑在功能时钟频率下（如 200MHz），验证 setup/hold 时序
- **Golden Signature 管理**：综合后网表的签名可能因 DFT 插入变，需要重新计算

### 典型测试模式

| 模式 | 说明 |
|------|------|
| **BIST_NORMAL** | 标准逻辑 BIST，伪随机模式全覆盖 |
| **BIST_PRBS7/15/23/31** | 特定 PRBS 多项式，用于 PHY 层面的误码率测试 |
| **BIST_LOOPBACK** | 内部回环：PAL_TX 输出直接环回给 PAL_RX |
| **BIST_SCAN** | 扫描链测试（stuck-at + transition delay） |

## 7. HWR — Hardware Register / Hard-Wired (硬件寄存器 / 硬连线)

### 全称
- **全称：** Hardware Register **或** Hard-Wired Register
- **别名：** Status Register (只读状态寄存器)

### 在 SerDes 中的角色

HWR 在芯片设计中指两种不同但相关的概念：

#### 概念一：硬连线只读寄存器 (Hard-Wired Register)

在 SerDes 的寄存器空间中，部分寄存器的值是**硬编码**的，不由软件配置，而是由硬件状态自动驱动：

```
APB 读地址 0x1C ──► 读 MUX ──► 输出数据总线
                         ▲
                         │
                    ┌────┴────┐
                    │  HWR    │ ← 值来自硬件线路，不是寄存器 flop
                    │ (硬连线) │
                    └────┬────┘
                         │
                    ┌────┴────┐
                    │ SerDes  │
                    │ 状态机   │── LOCK 状态
                    │ 监控     │── BER 计数
                    │ 逻辑     │── 温度/电压传感器
                    └─────────┘
```

**典型 HWR 示例：**

| 寄存器 | 位域 | 来源 | 说明 |
|--------|------|------|------|
| `HWR_LOCK_STATUS` | [0] | CDR 锁定检测器 | 1 = RX CDR 已锁定 |
| `HWR_BER_COUNTER` | [15:0] | CRC 错误计数器 | 递增计数接收到的 CRC 错误帧数 |
| `HWR_TEMP_SENSOR` | [7:0] | 片上温度传感器 | 当前结温（只读） |
| `HWR_CHIP_ID` | [31:0] | eFuse 读出值 | 芯片唯一 ID（上电加载） |
| `HWR_STRAP_VALUE` | [4:0] | 复位时采样的 strap 管脚 | 芯片配置模式（只读） |

**验证要点：**
- HWR 在复位后应立即反映硬件真实状态，而不是 0/未知
- 某些 HWR 有"清零条件"（如 BER 计数器需写 1 清除），需验证写清除逻辑
- Strap 类 HWR 只在复位释放瞬间采样，之后变化无效
- 状态类 HWR 在仿真中需配合驱动对应硬件条件来验证值是否正确

#### 概念二：硬件资源 (Hardware Resource)

在某些语境下 HWR 指硬件模块实例，如 "HWR block" 表示一个可重用的硬件加速器或硬件功能单元（如 CRC 计算器、加扰器），区别于软件实现。

## 8. DUMMY 块 (虚拟单元/模块)

### 在 SerDes 中的角色

"DUMMY" 在芯片设计中有多种含义，取决于上下文：

#### 1. DUMMY Filler Cell（物理设计）

在 SerDes 布局布线阶段，标准单元行中会有空隙，插入 **filler cell (dummy cell)** 来满足：

- **N-well 连续性**：避免 DRC 违例
- **密度规则**：Foundry 要求每层金属有最小密度（如 30%），否则 CMP 会导致凹陷
- **时钟树平衡**：在时钟路径上插入 dummy buffer 匹配延迟

```
┌──────┬──────┬────────┬──────┬──────┐
│ INV  │ NAND │  DUMMY │  DFF │  AND │
│      │      │ (填充) │      │      │
└──────┴──────┴────────┴──────┴──────┘
        标准单元行
```

#### 2. DUMMY Register（验证/DFT）

在寄存器域中故意预留的**保留位**，软件写入不影响功能，读出值未定义，用于：

- 地址空间对齐
- 后续功能扩展预留
- 兼容性考虑

```
Register 0x20: CFG_DATA [31:0]
  [31:24] DUMMY      — 保留（写忽略，读返回 0）
  [23:16] TX_GAIN    — 发送器增益
  [15:8]  RX_EQ      — 接收均衡器
  [7:0]   DRIVE_STR  — 驱动强度
```

#### 3. DUMMY Transaction（验证环境）

在 SerDes 验证中，**DUMMY transaction** 指无效/填充数据包：

- 用于填充带宽，保持链路连续
- 接收端收到后直接丢弃，不处理
- 测试链路在空闲状态下的行为

#### 4. DUMMY Load（模拟电路）

在 HPLL/DPLL 中，VCO 输出路径上插入的 dummy 负载单元，用于**匹配**关键路径的负载，保证时钟树延迟一致。

#### 验证要点
- Filler cell 不影响功能仿真，但 gate-level 仿真需确认无 timing 问题
- Dummy 寄存器的读写行为需在验证计划中明确（读返回 0？不定值？）
- Dummy transaction 的处理路径需验证（正常丢弃，不产生错误中断）

## 9. PRBS — Pseudo-Random Binary Sequence (伪随机二进制序列)

### 全称
- **全称：** Pseudo-Random Binary Sequence
- **常见阶数：** PRBS7, PRBS9, PRBS15, PRBS23, PRBS31

### 在 SerDes 中的角色

PRBS 是 SerDes **误码率测试**的核心信号源：

```
PRBS 发生器 (TX) ──► HPLL ──► 串行输出 ──► 通道 ──► RX ──► DPLL CDR ──► PRBS 校验器 (RX)
                        ↑                          ↑                        ↓
                   相同的 LFSR 多项式          通道损伤(噪声/ISI)       比较期望值与实际值
                                                                           ↓
                                                                     统计误码数 → BER
```

| 阶数 | 多项式 | 序列长度 | 应用场景 |
|------|--------|---------|---------|
| **PRBS7** | x⁷ + x⁶ + 1 | 127 bits | 8b/10b 编码链路（短序列，快速测试） |
| **PRBS9** | x⁹ + x⁵ + 1 | 511 bits | 一般 SerDes 测试 |
| **PRBS15** | x¹⁵ + x¹⁴ + 1 | 32,767 bits | 车载 SerDes 常用（覆盖典型通道损伤） |
| **PRBS23** | x²³ + x¹⁸ + 1 | 8,388,607 bits | 高速长链路测试 |
| **PRBS31** | x³¹ + x²⁸ + 1 | 2.1×10⁹ bits | 最坏情况（最长连 0/1 串，测试 CDR 保持能力） |

### 验证要点
- PRBS 发生器和校验器的 LFSR 多项式必须完全一致
- 初始种子 (seed) 可配置，两端必须同步
- 校验器需要帧同步机制（通常用 sync header 对齐）
- PRBS31 仿真时间极长（21 亿 bit），需用缩短序列或分段仿真

## 10. BER — Bit Error Rate (误码率)

### 全称
- **全称：** Bit Error Rate / Bit Error Ratio

### 定义

BER = (接收错误的 bit 数) / (总发送 bit 数)

| 等级 | BER | 含义 |
|------|-----|------|
| 光通信 | < 10⁻¹² | 每 10¹² bit 出错 < 1 bit |
| 车载 SerDes | < 10⁻¹² | 通常要求，视频流不卡顿 |
| PCIe Gen3+ | < 10⁻¹² | 同等级别 |
| 以太网 10GBASE-KR | < 10⁻¹² | 需 FEC 辅助 |
| 消费电子 | < 10⁻⁹ | 较低要求 |

### 在 SerDes 中的角色

SerDes 芯片在出厂测试时需要保证 BER < 10⁻¹²，验证中需关注：

- **PRBS 误码测试**：发送 PRBS 序列，接收端校验，统计误码数
- **眼图余量 (Eye Margin)**：BER 与眼高/眼宽直接相关
- **浴盆曲线 (Bathtub Curve)**：BER 随采样相位变化的曲线，用于确定最佳采样点
- 车载 SerDes 还需在 EMC 干扰下测试 BER（CISPR 25 标准）

---

## 11. CTLE — Continuous Time Linear Equalizer (连续时间线性均衡器)

### 全称
- **全称：** Continuous Time Linear Equalizer

### 在 SerDes 中的角色

高速信号经过 PCB 走线/同轴电缆后，高频分量衰减远大于低频（趋肤效应 + 介质损耗），导致码间干扰 (ISI)。CTLE 是 SerDes RX 前端的第一个均衡级：

```
RX± ──► ESD ──► CTLE ──► 采样器 ──► CDR (DPLL)

CTLE 内部:
  ┌───────────────┐
  │  差分放大器    │
  │  ┌─────────┐  │    增益 = gm * Z_load
  │  │ gm 级   │──┼──►  Z_load 中有零点/极点对:
  │  └─────────┘  │     零点的位置由源极退化电阻+电容设定
  │     ↑         │     增强高频增益，补偿信道损耗
  │   增益控制     │
  └───────────────┘
```

| 参数 | 说明 | 验证要点 |
|------|------|---------|
| **DC 增益** | 低频增益（通常 0~6dB） | PVT 下一致性 |
| **高频增益** | 在 Nyquist 频率处的增益（可达 12~20dB） | 补偿量是否足够 |
| **零点频率** | 均衡开始增强的频率点 | 需匹配信道特性 |
| **功耗** | 模拟电路电流 | IR drop 分析 |

## 12. DFE — Decision Feedback Equalizer (判决反馈均衡器)

### 全称
- **全称：** Decision Feedback Equalizer

### 在 SerDes 中的角色

DFE 是 RX 中的第二级均衡器，位于 CTLE 之后、CDR 之前。它利用**已经判决的历史 bit** 来消除当前 bit 的后标干扰 (post-cursor ISI)：

```
CTLE 输出 ──► ∑ ──► 判决器 ──► 恢复数据
               ▲                  │
               │                  ▼
               │            ┌──────────┐
               │            │  DFE     │
               │            │ 抽头系数  │  h1, h2, ..., hN
               │            │ (自适应)  │
               ▲            └──────────┘
               │                  │
         ┌─────┴─────┐           │
         │  加权求和  │◄──────────┘
         │ h1×D₁ +   │
         │ h2×D₂ +   │  D₁ = 前 1 个 bit
         │ ...       │  D₂ = 前 2 个 bit
         └───────────┘
```

| 抽头数 | SerDes 速率 | 典型值 |
|--------|------------|--------|
| 1~5 taps | ≤ 6 Gbps | 车载 SerDes 常用 |
| 5~15 taps | 10~28 Gbps | 高速 SerDes |
| 15+ taps | ≥ 56 Gbps | PAM4 需要 |

### 验证要点
- DFE 抽头系数的自适应收敛速度
- 错误传播 (Error Propagation)：如果历史判决错误，可能引发连续错误
- DFE 在长连 0/1 下的表现（无信号跳变时自适应可能发散）

## 13. SSC — Spread Spectrum Clocking (扩频时钟)

### 全称
- **全称：** Spread Spectrum Clocking

### 在 SerDes 中的角色

SSC 将 HPLL 的输出时钟以低频（通常 30~33 kHz）进行频率调制（±5000 ppm），将时钟能量扩散到更宽的频谱上，降低 EMI：

```
无 SSC:       有 SSC (调制后):
幅度│               幅度│
    │▓▓▓▓▓▓▓▓▓          │  ░░░░░░░░░░░░░░░░░
    │▓▓▓▓▓▓▓▓▓          │ ░░░░░░░░░░░░░░░░░░░
    │▓▓▓▓▓▓▓▓▓          │░░░░░░░░░░░░░░░░░░░░░
    └────────► 频率      └────────────────────► 频率
    窄带高峰              宽带低峰（EMI 降低 ~3~6dB）
```

| 参数 | 典型值 | 说明 |
|------|--------|------|
| **调制频率 (Fmod)** | 30~33 kHz | 低于音频范围，不产生可闻噪声 |
| **调制深度 (δ)** | −5000 ppm (down-spread) | 仅向低频方向扩 |
| **调制波形** | 三角波 / 赫尔希波 (Hershey-kiss) | 三角波最简单，赫尔希波 EMI 效果更好 |

### 验证要点
- SSC 调制深度和频率必须在协议允许范围内（如 PCIe 要求 −5000~0 ppm）
- SSC 对 CDR 的影响：接收端 DPLL 必须能跟踪 SSC 调制，不产生失锁
- SSC 调制波形的频谱验证（峰值 EMI 降低量）

## 14. PI — Phase Interpolator (相位插值器)

### 全称
- **全称：** Phase Interpolator

### 在 SerDes 中的角色

PI 是 DPLL CDR 中的关键组件，在 DCO 的多相位时钟之间进行插值，产生精细可调的采样相位：

```
DCO 输出:  CLK_0° ──►│          │
           CLK_45° ──►│          │
           CLK_90° ──►│   PI     │──► 采样时钟 (相位 = CLK_0° + α × 45°)
           CLK_135°──►│ (插值)   │     α = 数字控制字
           ...        │          │
           CLK_315°──►│          │
                      └──────────┘
```

| 参数 | 说明 |
|------|------|
| **分辨率** | 最小可调相位步进（如 0.5ps @ 10Gbps = UI/200） |
| **线性度 (INL/DNL)** | 插值精度，非线性和导致 CDR 抖动 |
| **工作频率** | 需覆盖 SerDes 所有速率 |

### 验证要点
- PI 的 INL/DNL 指标直接决定 CDR 残余抖动
- PI 控制码的单调性验证（码值 +1，相位必须单调向一个方向变化）
- PI 在 PVT 下的相位精度

## 15. CDR — Clock Data Recovery (时钟数据恢复)

### 全称
- **全称：** Clock Data Recovery

### 在 SerDes 中的角色

CDR 是 SerDes 接收端的核心，从串行数据流中**提取出时钟并重定时数据**。在车载 SerDes 中通常用 DPLL 实现：

```
串行数据 ──► 边沿采样器 ──┐
                          ├──► BBPD ──► 数字滤波器 ──► PI ──► 恢复时钟
串行数据 ──► 数据采样器 ──┘                            │
                                                       ▼
                                                 重定时数据输出
```

**CDR 工作模式：**

| 模式 | 说明 | 条件 |
|------|------|------|
| **频率捕获** | FLL 辅助，将 DCO 拉到目标频率 | 上电/唤醒 |
| **相位跟踪** | BBPD + 数字环路滤波，跟踪输入相位变化 | 正常接收数据 |
| **保持 (Hold)** | 无数据跳变时，DCO 维持上次频率 | 长连 0/1 |
| **失锁恢复** | 检测到失锁后重新进入频率捕获 | 输入信号中断后恢复 |

### 验证要点
- 锁定时间：从有效数据出现到 CDR 锁定，需 < 协议规定
- 频偏容忍：输入数据频率与本地参考的偏差容忍度（如 ±300 ppm）
- 抖动传递函数 (Jitter Transfer, JTRAN)：CDR 对不同频率抖动的响应
- 抖动容限 (Jitter Tolerance, JTOL)：CDR 能容忍的最大输入抖动
- 长连 0/1 测试：连续相同 bit 后，数据恢复仍正确

---

## 16. 8b/10b 编码

### 全称
- **全称：** 8-bit to 10-bit encoding
- **发明：** IBM (Albert Widmer, Peter Franaszek)

### 在 SerDes 中的角色

将每 8 bit 数据编码为 10 bit 编码字（码字），目的是：

```
8b 数据: 10101010 (0xAA)
       │
       ▼
    编码器 (基于 RD - Running Disparity)
       │
       ▼
10b 码字: 1010110100 (0x2B4)
    ├── 保证 DC 平衡：0 和 1 的数量差 ≤ 2
    ├── 保证足够多的跳变（利于 CDR 锁定）
    └── 包含专用 K 码（控制字，用于帧同步/对齐）
```

| 好处 | 说明 |
|------|------|
| **DC 平衡** | 交流耦合通道不会因 DC 漂移导致误码 |
| **时钟嵌入** | 足够多的跳变让 CDR 持续锁定 |
| **专用 K 码** | 同步头、帧起始/结束、空闲标志 |
| **检错** | 非法码字可检测（单 bit 错误检出的概率高） |

**K 码 (控制码)：**

| K 码 | 名称 | 用途 |
|------|------|------|
| K28.5 (0xBC) | COM | 同步头 / 对齐字 |
| K28.1 (0x3C) | SDP | 数据包起始 |
| K28.2 (0x5C) | SHP | 控制包起始 |
| K28.0 (0x1C) | SKP | 时钟间隙填充 |
| K29.7 (0xFD) | END | 包结束 |

### 验证要点
- 编码器与解码器的映射表一致性
- RD (Running Disparity) 计算正确性，编码前后的 RD 必须匹配
- 非法 K 码组合的检测与错误处理
- 编码效率损失：8b/10b = 80% 有效带宽（10Gbps 实际数据 8Gbps）

## 17. 加扰器 / 解扰器 (Scrambler / Descrambler)

### 全称
- **全称：** Scrambler / Descrambler

### 在 SerDes 中的角色

加扰器对发送数据进行**随机化**处理，避免长时间重复图案导致的 EMI 峰值和 CDR 失锁：

```
TX 方向:
  原始数据 ──► XOR ──► 加扰数据 ──► 串行器 ──► HPLL ──► TX±
                ▲
                │
         LFSR (与 PRBS 相同的结构)

  解扰器 (RX) 与加扰器同步后，通过 XOR 恢复原始数据。

  自同步加扰器 (Self-Sync Scrambler):
    加扰:  D_out = D_in XOR D_out[t-n] XOR D_out[t-m]
    解扰:  D_out = D_in XOR D_in[t-n] XOR D_in[t-m]
    (不需要帧同步，数据处理即同步)
```

| 加扰器类型 | 同步方式 | 优点 | 缺点 |
|-----------|---------|------|------|
| **自同步** | 基于历史数据 | 无需帧同步，即插即用 | 误码会倍数传播 |
| **帧同步** | 基于 sync header | 误码不传播 | 需帧同步机制 |

---

## 18. CRC — Cyclic Redundancy Check (循环冗余校验)

### 在 SerDes 中的角色

车载 SerDes 传输视频/控制数据时，CRC 用于检测数据传输错误：

```
TX: 数据 ──► CRC 计算器 ──► 将 CRC 附加在数据帧尾部发送
RX: 数据 ──► CRC 计算器 ──► 比较计算结果与收到的 CRC 值
                │
                ▼
         匹配 → 数据正确
         不匹配 → 报告 CRC 错误，可能触发重传或错误掩蔽
```

**车载 SerDes 常见 CRC 多项式：**

| CRC | 多项式 | 应用 |
|-----|--------|------|
| CRC-8 | x⁸ + x² + x + 1 | 控制帧校验 |
| CRC-16 | x¹⁶ + x¹² + x⁵ + 1 | 数据帧校验 |
| CRC-32 | x³² + x²⁶ + x²³ + ... + 1 | 高级数据校验 |

### 验证要点
- CRC 算法 RTL 与参考模型的一致性（用 Python/C 参考模型交叉比对）
- CRC 错误注入测试：翻转帧中任一 bit，RX CRC 必须检测到错误
- CRC 多帧/连续帧的累积校验

## 19. 眼图 (Eye Diagram)

### 在 SerDes 中的角色

眼图是评估 SerDes **信号质量**的核心可视化工具。将多次采样波形叠加显示：

```
              ▒▒▒▒▒▒▒▒▒▒▒▒
     ▒▒▒▒             ▒▒▒▒         ▒▒▒▒
 ▒▒▒▒    ▒▒▒▒     ▒▒▒▒    ▒▒▒▒ ▒▒▒▒
        ▒▒▒▒             ▒▒▒▒
        ▒▒▒▒     ▒▒▒▒    ▒▒▒▒
 ▒▒▒▒    ▒▒▒▒ ▒▒▒▒    ▒▒▒▒
     ▒▒▒▒      ▒▒▒▒
              ▒▒▒▒▒▒▒▒▒▒▒▒

   ← 眼高 (Eye Height) →    ← 眼宽 (Eye Width) →   ← 抖动 (Jitter) →
```

| 眼图参数 | 说明 | 影响 |
|---------|------|------|
| **眼高 (Eye Height)** | 信号开口的垂直幅度 (mV) | 噪声容限 |
| **眼宽 (Eye Width)** | 信号开口的水平宽度 (UI) | 抖动容限 |
| **抖动 (Jitter)** | 边沿的随机/确定性偏离 | BER |
| **占空比失真 (DCD)** | 高/低脉冲宽度不对称 | 低频抖动 |
| **Q 因子** | 眼图质量的信噪比度量 | BER 的关联指标 |

**仿真验证：**
- 在 AMS 仿真中通过 PRBS 激励跑数百到数千个 UI
- 提取眼高/眼宽统计值，与 spec 对比
- 用 Verdi/SimVision 等工具查看眼图

## 20. 浴盆曲线 (Bathtub Curve)

### 在 SerDes 中的角色

浴盆曲线表示 BER 随采样相位变化的曲线，是评估 **CDR 采样点安全性**的标准方法：

```
BER
  ▲
  │ ▓▓▓▓▓▓▓▓▓▓                          ▓▓▓▓▓▓▓▓▓▓
  │         ▓▓▓▓▓▓                   ▓▓▓▓▓▓
  │              ▓▓▓▓            ▓▓▓▓
  │                  ▓▓▓▓    ▓▓▓▓
  │                       ▓▓▓▓                    ← BER < 10⁻¹² 区域
  └─────────────────────────────────────────► 采样相位
    ← 左眼沿          最佳采样点        右眼沿 →
    ←   眼宽     →
```

| 参数 | 说明 |
|------|------|
| **眼宽 @ BER=10⁻¹²** | 浴盆曲线在 10⁻¹² 水平线处的开口宽度 |
| **眼宽余量** | 眼宽与 1 UI 的比值（如 0.4 UI @ BER=10⁻¹²） |
| **最佳采样点** | 浴盆底部最平坦处，CDR 应锁定于此 |

## 21. 抖动 (Jitter) — SJ / RJ / DJ / TJ

### 在 SerDes 中的角色

抖动是 SerDes 信号边沿偏离理想位置的时域偏差，直接影响 BER：

```
理想时钟:  ____|____|____|____|____|____|____
              ↑ ↑ ↑ ↑ ↑ ↑ ↑ ↑ → 理想边沿

带抖动:    ____|_ _|_ _|_ _|___|_ _|_ _|____
              ↑ ↑ ↑ ↑ ↑ ↑ ↑ ↑ → 边沿位置变化
              ←   抖动区间   →
```

| 抖动类型 | 来源 | 特点 |
|---------|------|------|
| **RJ (Random Jitter)** | 热噪声、散粒噪声 | 高斯分布，无界，RMS 值 |
| **SJ (Sinusoidal Jitter)** | 电源噪声、EMI | 正弦调制，有界，峰值 |
| **DJ (Deterministic Jitter)** | ISI、DCD | 有界，双峰分布 |
| **TJ (Total Jitter)** | RJ + DJ | 总抖动，@ BER=10⁻¹² |

**验证要点：**
- 抖动注入测试：在仿真中注入不同类型/幅度的抖动，验证 CDR 跟踪能力
- 抖动容限 (JTOL) 模板：绘制 JTOL 曲线，验证是否在所有频率上超过 spec 要求
- 抖动传递 (JTRAN)：CDR 对抖动的滤波特性（高频抖动应被抑制）

## 22. LVDS / CML / HCS — I/O 电平标准

### 在 SerDes 中的角色

串行数据输出驱动器采用不同的 I/O 电平标准，取决于速率和应用：

```
LVDS (Low-Voltage Differential Signaling):
  ┌──────┐    100Ω           ┌──────┐
  │ TX   │───►  ├───►  ────│ RX   │
  │      │───►  └───►  ────│      │
  └──────┘   偏置 ~1.2V    └──────┘
  Swing: 350mV, 常用 ≤ 2Gbps

CML (Current Mode Logic):
  ┌──────┐    VDD             ┌──────┐
  │ TX   │───►  █ 50Ω ──►    │ RX   │
  │      │───►  █ 50Ω ──►    │      │
  └──────┘    VDD             └──────┘
  Swing: 400~800mV, 常用 ≥ 3Gbps

HCS (High-Speed Current Steering):
  ┌──────┐     ┌─ 50Ω ─► VTT   ┌──────┐
  │ TX   │───►  ├─ 50Ω ─► VTT   │ RX   │
  │      │───►  ├─ 50Ω ─► VTT   │      │
  └──────┘     └─ 50Ω ─► VTT   └──────┘
  Swing: 200~400mV, 超高速 SerDes
```

| 标准 | 摆幅 | 速率范围 | 功耗 | 车载 SerDes 常用 |
|------|------|---------|------|----------------|
| **LVDS** | 350mV | ≤ 3.125 Gbps | 低 | 早期车载链路 |
| **CML** | 400~800mV | ≤ 28 Gbps | 中 | 中高速 SerDes |
| **HCS** | 200~400mV | ≥ 28 Gbps | 低 | 超高速 SerDes |

## 23. APB / AHB / AXI — 总线接口

### 在 SerDes 中的角色

这些是 SoC 内部总线，用于 MCU 访问 SerDes 的控制寄存器：

```
MCU (CPU)
  │
  ├── APB ──► 低速外设 (GPIO, eFuse, I2C, SPI 配置接口)
  ├── AHB ──► 中速 (DMA, SRAM)
  └── AXI ──► 高速数据路径 (视频流数据)
```

在 SerDes 芯片中：
- **APB**：配置 SerDes 各模块寄存器（HPLL trim、EQ 系数、CDR 带宽）
- **AHB/AXI**：传输视频流数据（摄像头数据 → SerDes TX）

## 24. TAP — Test Access Port (测试访问端口) — IEEE 1149.1

### 在 SerDes 中的角色

TAP 是 JTAG 的核心控制器，一个 16 状态的有限状态机，控制所有 TDR 的操作：

```
        ┌─────────┐
        │       0 │
        │ TL-Reset│──────────────────────┐
        └────┬────┘                      │
             │ 1                         │
        ┌────▼────┐                      │
        │ Run-Test│                      │
        │   /Idle │──────────────────┐   │
        └────┬────┘                  │   │
             │ ？？(选择性)          │   │
        ┌────▼────┐                  │   │
        │ Select-DR│    Select-IR     │   │
        └────┬────┘   └──────┬───────┘   │
             ▼                ▼           │
       Capture-DR ← ─  Capture-IR        │
             ▼                ▼           │
         Shift-DR ← ─  Shift-IR          │
             ▼                ▼           │
         Exit1-DR  ← ─  Exit1-IR         │
             ▼                ▼           │
  ┌─── Pause-DR ← ─  Pause-IR ──┐        │
  └─── Exit2-DR ← ─ Exit2-IR ───┘        │
             ▼                ▼           │
        Update-DR ← ── Update-IR ─────────┘
             ▼                ▼
          (回到 Run-Test/Idle)
```

**TAP 接口信号 (5 线 JTAG)：**
- **TCK** — 测试时钟
- **TMS** — 测试模式选择（状态转移控制）
- **TDI** — 测试数据输入
- **TDO** — 测试数据输出
- **TRST** (可选) — 测试复位

## 25. MIPI — Mobile Industry Processor Interface

### 在 SerDes 中的角色

MIPI 是移动/车载 SoC 与摄像头/显示器之间的标准化接口协议。车载 SerDes 芯片通常一端接 MIPI CSI (摄像头) 或 DSI (显示器)：

```
摄像头 SoC ──► MIPI CSI ──► TX SerDes ──► 同轴电缆 ──► RX SerDes ──► MIPI CSI ──► 显示器/域控制器
                     │                                    │
                 4-lane MIPI                          重建的 4-lane MIPI
               (每 lane ≤ 1.5Gbps)                    (每 lane ≤ 1.5Gbps)
```

| 协议 | 全称 | SerDes 用途 |
|------|------|-------------|
| **MIPI CSI-2** | Camera Serial Interface 2 | 摄像头图像数据输入/输出 |
| **MIPI DSI** | Display Serial Interface | 显示器数据输入/输出 |
| **MIPI D-PHY** | MIPI 物理层 | 1 lane 最多 4 lanes，每 lane ≤ 1.5Gbps |
| **MIPI C-PHY** | MIPI 物理层（三线） | 每 wire 最高 2.5Gsps |

### 验证要点
- MIPI CSI/DSI 协议解码与 SerDes 帧格式的映射
- LP (Low Power) 模式与 HS (High Speed) 模式切换
- MIPI 协议时序要求（HS 准备时间、LP 状态机）

## 术语速查表

| 术语 | 中文 | 一句话解释 | 在 SerDes 中 |
|------|------|-----------|-------------|
| **BER** | 误码率 | 出错的 bit 数 / 总 bit 数 | SerDes 链路质量的最终度量 |
| **CDR** | 时钟数据恢复 | 从数据流中提取时钟 | RX 核心，常用 DPLL 实现 |
| **CRC** | 循环冗余校验 | 数据帧的校验和 | 检测传输错误 |
| **CTLE** | 连续时间线性均衡器 | 模拟 EQ，补偿信道高频损耗 | RX 第一级，处理 ISI |
| **DFE** | 判决反馈均衡器 | 用历史 bit 消除后标干扰 | RX 第二级，处理长尾 ISI |
| **DLL** | 延迟锁相环 | 延迟锁定环路，用于时钟对齐 | SerDes 中较少用，MIPI 有 |
| **DCO** | 数字控制振荡器 | 数字控制的时钟发生器 | DPLL CDR 的核心 |
| **DCD** | 占空比失真 | 高/低脉冲宽度不对称 | 导致确定性抖动的一种 |
| **FEC** | 前向纠错 | 发射端加冗余，接收端纠错 | 超高速或高可靠性链路用 |
| **FLL** | 频率锁定环路 | 粗调频率的辅助环路 | CDR 频率捕获阶段 |
| **ISI** | 码间干扰 | 邻近 bit 相互干扰的效应 | 高速 SerDes 的主要损伤 |
| **JTOL** | 抖动容限 | CDR 能容忍的最大抖动 | CDR 捕获能力的关键指标 |
| **JTRAN** | 抖动传递 | CDR 对抖动的滤波响应 | 反映 CDR 带宽特性 |
| **LOS** | 信号丢失 | RX 检测到无输入信号 | 告警指示，触发链路重训练 |
| **LFSR** | 线性反馈移位寄存器 | 产生伪随机序列的电路 | PRBS 发生器、加扰器的核心 |
| **MISR** | 多输入签名寄存器 | 将多路输出压缩为一个签名 | BIST 结果校验 |
| **PFD** | 鉴频鉴相器 | 检测频率和相位差 | HPLL 和 DPLL 都用 |
| **PI** | 相位插值器 | 在多相时钟间精细插值 | DPLL CDR 的相位调整 |
| **PRBS** | 伪随机二进制序列 | 确定性随机序列 | SerDes 测试的标准激励 |
| **SER/DES** | 串行器/解串器 | 并→串 / 串→并转换 | SerDes 芯片的核心功能 |
| **SSC** | 扩频时钟 | 调制时钟以降低 EMI | 车载 SerDes EMI 合规 |
| **TAP** | 测试访问端口 | JTAG 的 16 状态 FSM | 控制所有测试功能 |
| **TG** | 测试模式 | 芯片测试/调试工作模式 | 区别于 Normal 模式 |
| **GMSL** | 千兆多媒体串行链路 | Maxim 的车载 SerDes 协议 | 车内摄像头/显示链路 |
| **FPD-Link** | 平板显示链路 | TI 的车载 SerDes 协议 | 同上，与 GMSL 竞争 |
