# 随机化约束

## 1. 随机化概述

随机化（Randomization）是SystemVerilog验证方法学的核心支柱之一。通过随机化，可以自动生成大量不同的激励场景，大幅提高功能覆盖率，发现手动测试难以覆盖的边界条件和隐藏缺陷。

**随机化的重要性：**

- **提高覆盖率**：随机激励能探索到手写测试遗漏的场景组合
- **减少测试编写时间**：只需定义约束空间，而非逐一枚举测试向量
- **发现隐藏缺陷**：真正的随机值能触发设计者未预料到的边界行为
- **可复现性**：通过种子（seed）可精确复现失败的随机场景

> [!tip] 随机化 vs 定向测试
> 现代验证方法学采用"约束随机验证"（Constrained Random Verification, CRV）与定向测试相结合的策略。随机化覆盖广度，定向测试覆盖已知的边界和关键场景。

## 2. rand 和 randc 关键字

SystemVerilog提供两种随机化修饰符，用于声明类的属性为随机变量。

### 2.1 rand — 标准随机

`rand` 每次随机化时为变量生成一个新的随机值，值的分布服从均匀分布（或由约束指定的分布）。

```verilog
class Packet;
  rand bit [7:0]  length;
  rand bit [31:0] payload[];
  rand bit [1:0]  priority_level;
endclass
```

### 2.2 randc — 周期随机

`randc`（random cyclic）在所有可能的值都被生成一遍之前，不会重复。适用于需要遍历所有值的场景。

```verilog
class TestSeq;
  randc bit [3:0] opcode;  // 0~15每个值恰好出现一次后才重复
endclass
```

### 2.3 rand 与 randc 的区别

| 特性 | rand | randc |
|------|------|-------|
| 值重复 | 可能立即重复 | 遍历所有值后才重复 |
| 适用类型 | 所有整数/位向量 | 仅限整数/枚举 |
| 典型用途 | 通用随机激励 | 遍历测试、地址轮转 |
| 性能 | 更快 | 需维护已生成值的历史 |

> [!warning] randc 限制
> - `randc` 只能用于整数类型和枚举类型，不能用于实数或队列
> - `randc` 变量不能在 `solve...before` 中被约束求解顺序影响

## 3. constraint 块语法

约束块定义了随机变量合法值的范围。求解器（Solver）会在约束空间内随机选取满足所有约束的值。

### 3.1 基本语法

```verilog
class Transaction;
  rand bit [7:0] addr;
  rand bit [31:0] data;
  rand bit        write;

  constraint valid_range {
    addr inside {[0:127]};          // addr范围: 0~127
    data inside {32'h0, 32'hFFFF_FFFF};  // data只能是0或全1
  }
endclass
```

### 3.2 条件约束

```verilog
class BusTransaction;
  rand bit [2:0] burst_len;
  rand bit [31:0] addr;
  rand bit        is_aligned;

  constraint burst_c {
    if (is_aligned) {
      addr[1:0] == 2'b00;          // 对齐访问时地址4字节对齐
      burst_len inside {[1:4]};
    } else {
      burst_len == 1;              // 非对齐访问只能单次传输
    }
  }
endclass
```

### 3.3 权重约束与分布

```verilog
class Config;
  rand bit [1:0] mode;

  constraint mode_dist {
    mode dist {0 := 40, 1 := 30, 2 := 20, 3 := 10};
    // mode=0的概率约40%, mode=3的概率约10%
  }
endclass
```

`:=` 表示每个值的权重独立计算；`:\/` 表示将权重平均分配给范围内所有值。

### 3.4 solve...before（求解顺序）

```verilog
class OrderedSolve;
  rand bit [7:0] x, y;

  constraint relation {
    x < y;
    solve x before y;  // 先确定x，再确定y，避免求解器反复回溯
  }
endclass
```

### 3.5 外部约束

约束可以在类内部定义，也可以在类外部定义，便于灵活扩展。

```verilog
class BaseItem;
  rand bit [7:0] id;
  // 只声明，不定义体
  constraint soft_id;
endclass

// 外部定义约束体
constraint BaseItem::soft_id {
  id inside {[1:100]};
}
```

### 3.6 软约束（soft constraint）

软约束优先级低于硬约束，当硬约束冲突时，软约束被自动忽略。

```verilog
class Item;
  rand int val;

  constraint c_soft {
    soft val inside {[1:10]};  // 默认取1~10，但可被硬约束覆盖
  }
endclass

Item item = new();
item.randomize() with { val > 100; };  // 硬约束生效，软约束被忽略
```

### 3.7 约束继承与覆盖

```verilog
class Base;
  rand bit [7:0] data;

  constraint c_base {
    data inside {[0:127]};
  }
endclass

class Child extends Base;
  // 覆盖父类约束
  constraint c_base {
    data inside {[0:63]};
  }
endclass
```

## 4. 约束求解和冲突处理

### 4.1 求解机制

SystemVerilog的约束求解器基于约束满足问题（CSP）算法，在所有约束的交集空间内随机选取一个合法解。求解器保证：

- 所有硬约束（hard constraint）必须满足
- 软约束在不冲突的情况下尽量满足
- 随机种子决定具体选取哪个解

### 4.2 约束冲突

当约束空间为空（无合法解）时，`randomize()` 返回0（失败），随机变量保持原值不变。

```verilog
class ConflictExample;
  rand bit [3:0] val;

  constraint c1 { val > 10; }  // val > 10
  constraint c2 { val < 5;  }  // val < 5 → 冲突！4位宽最大15，无交集
endclass

ConflictExample obj = new();
if (!obj.randomize()) begin
  $display("随机化失败：约束冲突");
end
```

### 4.3 调试约束冲突

```verilog
// 方法一：使用randomize()的返回值
assert(obj.randomize()) else
  $error("randomize failed: %s", obj.randomize());

// 方法二：使用inline constraint逐步排查
if (!obj.randomize() with { val > 10; }) begin
  $warning("约束冲突");
end
```

> [!warning] 约束冲突陷阱
> - 约束求解器不会报错，仅返回0，务必检查返回值
> - 使用 `randomize() with {}` 添加内联约束时，注意不要与类内约束冲突
> - `randc`变量的约束更容易产生隐性冲突（值域有限）

## 5. 随机化方法

### 5.1 randomize()

`randomize()` 是核心方法，触发约束求解并更新所有 `rand`/`randc` 变量。

```verilog
class MyItem;
  rand bit [7:0] a;
  rand bit [7:0] b;

  constraint c_sum { a + b < 256; }
endclass

MyItem item = new();
int success;
success = item.randomize();  // 返回1表示成功，0表示失败
$display("a=%0d, b=%0d", item.a, item.b);
```

**randomize() with 内联约束：**

```verilog
// 只随机化a，b保持不变（使用rand变量名作为with参数）
success = item.randomize(a);
$display("only a randomized: a=%0d, b=%0d", item.a, item.b);

// 添加额外约束
success = item.randomize() with { a > 100; b < 50; };
```

### 5.2 pre_randomize() 和 post_randomize()

这两个回调函数在 `randomize()` 前后自动调用，用于执行额外逻辑。

```verilog
class ConfigPacket;
  rand bit [7:0] payload[];
  rand int unsigned pkt_len;

  // 随机化前：根据pkt_len预分配数组
  function void pre_randomize();
    $display("pre_randomize: 准备随机化...");
  endfunction

  // 随机化后：执行后处理
  function void post_randomize();
    $display("post_randomize: pkt_len=%0d", pkt_len);
    // 可在此计算派生字段
    foreach (payload[i])
      payload[i] = payload[i] ^ 8'hFF;  // 后处理：翻转
  endfunction
endclass
```

### 5.3 pre/post_randomize 的执行顺序

```
randomize()
  ├── pre_randomize()     // 自动调用，可设置约束前的上下文
  ├── 约束求解
  ├── 更新rand/randc变量
  └── post_randomize()    // 自动调用，可处理派生数据
```

## 6. 内建约束函数

SystemVerilog提供多个系统函数用于生成随机值，可在约束内部使用。

### 6.1 $urandom — 无符号32位随机

`$urandom()` 返回一个32位无符号随机值，每次调用产生独立的随机数。

```verilog
// 基本用法
bit [31:0] rnd_val = $urandom();

// 带种子：相同种子产生相同序列，用于复现
bit [31:0] val1 = $urandom(12345);  // 首次调用
bit [31:0] val2 = $urandom(12345);  // 相同种子 → val2 == val1
bit [31:0] val3 = $urandom();       // 不带种子 → 真随机

// 截取低位获取指定位宽的随机值
bit [7:0]  byte_rnd = $urandom();       // 自动截取低8位
bit [15:0] word_rnd = $urandom() & 16'hFFFF;  // 显式掩码

// 在测试中用于生成随机地址/数据
initial begin
  bit [31:0] base_addr = 32'h1000_0000;
  repeat (20) begin
    bit [31:0] offset = $urandom() % 256;  // 0~255的随机偏移
    $display("addr = 0x%08h", base_addr + offset);
  end
end
```

> [!warning] $urandom 种子行为
> - 不带参数的 `$urandom()` 使用仿真器自动分配的种子
> - 带参数时，参数是**种子值**而非范围，注意与 `$urandom_range` 的区别
> - 种子参数只在**首次调用**时生效，后续调用使用内部状态推进

### 6.2 $urandom_range — 指定范围随机

`$urandom_range(max, min=0)` 生成 `[min, max]` 闭区间内的均匀分布随机值。

```verilog
// 双参数形式：[min, max]
int val = $urandom_range(0, 9);     // 0~9
int val2 = $urandom_range(10, 20);  // 10~20

// 单参数形式：[0, max]
int val3 = $urandom_range(9);       // 等价于 $urandom_range(0, 9)

// 常见应用场景
initial begin
  // 随机延迟
  int delay = $urandom_range(5, 20);
  #delay;

  // 随机选择操作类型
  int op = $urandom_range(0, 3);  // 0=READ, 1=WRITE, 2=IDLE, 3=BURST

  // 随机生成错误注入位置
  int err_pos = $urandom_range(0, payload.size()-1);
  payload[err_pos] = ~payload[err_pos];
end
```

**与约束中 inside 的对比：**

```verilog
// 方式一：$urandom_range（过程代码中）
int a = $urandom_range(0, 100);

// 方式二：constraint + inside（类随机化中）
class Item;
  rand int a;
  constraint c_a { a inside {[0:100]}; }
endclass
```

> [!tip] 选择建议
> - 在 class 内部定义随机变量 → 用 `rand` + `randomize()` + constraint
> - 在 task/initial 等过程代码中生成随机值 → 用 `$urandom_range`
> - 需要变量间关联约束 → 只能用 `randomize()`

### 6.3 $srandom — 设置随机种子

```verilog
// 手动设置当前线程的随机种子
process::self.srandom(12345);

// 在class内部设置种子
class SeededItem;
  rand bit [7:0] data;
  function new(int seed);
    this.srandom(seed);  // 设置此对象的随机种子
  endfunction
endclass
```

### 6.4 约束中的高级模式

#### foreach 约束（数组遍历约束）

```verilog
class ArrayConstraint;
  rand bit [7:0] arr[10];

  constraint c_arr {
    foreach (arr[i]) {
      arr[i] inside {[0:100]};         // 每个元素范围
      if (i > 0) arr[i] > arr[i-1];   // 递增
    }
  }
endclass

// 动态数组大小约束
class DynArray;
  rand bit [7:0] data[];
  constraint c_size {
    data.size() inside {[1:16]};        // 数组长度1~16
    foreach (data[i]) data[i] < 128;   // 每个元素 < 128
  }
endclass
```

#### unique 约束（唯一性约束）

```verilog
class UniqueTest;
  rand bit [3:0] ids[5];

  constraint c_unique {
    unique {ids};  // 5个ID互不相同
  }
endclass

// 实际应用：随机分配不重复的通道号
class ChannelAlloc;
  rand bit [2:0] ch_sel[4];

  constraint c_ch {
    unique {ch_sel};              // 4个通道号不重复
    foreach (ch_sel[i])
      ch_sel[i] inside {[0:7]};  // 通道号范围0~7
  }
endclass
```

#### implication 约束（蕴含约束 → 操作符）

```verilog
// → 操作符等价于 if...then，更简洁
class ImplTest;
  rand bit       is_write;
  rand bit [7:0] wr_data;
  rand bit [7:0] rd_data;

  constraint c_impl {
    is_write  -> wr_data inside {[0:200]};   // 写模式时数据范围
    !is_write -> rd_data == 8'hFF;           // 读模式时返回固定值
  }
endclass
```

#### bidirectional 约束（双向求解）

```verilog
// 约束求解器是双向的：可以从任一变量推导另一变量
class BidirTest;
  rand bit [7:0] x, y;

  constraint c_xy {
    x + y == 100;  // 给定x可求y，给定y可求x
  }
endclass

BidirTest obj = new();
obj.randomize() with { x == 30; };  // y自动为70
obj.randomize() with { y == 80; };  // x自动为20
```

### 6.5 $urandom_range 在约束中的使用

> [!warning] 注意
> `$urandom_range` 是系统函数，**不能**直接出现在 constraint 块中作为约束表达式。约束中应使用 `inside` 语法来表达范围。

```verilog
// ❌ 错误：constraint中不能调用$urandom_range
// constraint c_bad { val == $urandom_range(0, 100); }

// ✅ 正确：在constraint中使用inside
class CorrectWay;
  rand bit [7:0] val;
  constraint c_good { val inside {[0:100]}; }
endclass

// ✅ $urandom_range 用于过程代码
initial begin
  repeat (100) begin
    int addr = $urandom_range(0, 1023);
    int data = $urandom_range(0, 255);
    bus_write(addr, data);
  end
end
```

### 6.6 随机数生成方式对比

| 函数/方法 | 位宽 | 可约束 | 线程安全 | 用途 |
|-----------|------|--------|----------|------|
| `$urandom` | 32位 | 否 | 每线程独立种子 | 独立随机值，可做种子 |
| `$urandom_range` | 可变 | 否 | 每线程独立种子 | 过程代码中的范围随机 |
| `rand` + `randomize()` | 可变 | 是 | 对象级 | 约束随机验证主力 |
| `$random` | 32位有符号 | 否 | 全局共享 | **不推荐**，仅兼容旧代码 |

> [!tip] 优先使用randomize()
> `$urandom` 和 `$urandom_range` 生成独立随机值，不参与约束求解。验证中应优先使用 `randomize()` 配合约束块，确保变量间关联性。`$urandom_range` 适合在 task/test 等过程代码中快速生成随机延迟、地址偏移等。

## 7. 与UVM的集成

### 7.1 uvm_sequence_item 的随机化

UVM的事务对象通常继承自 `uvm_sequence_item`，天然支持随机化。

```verilog
class my_transaction extends uvm_sequence_item;
  `uvm_object_utils(my_transaction)

  rand bit [7:0]  cmd;
  rand bit [31:0] addr;
  rand bit [31:0] data;
  rand bit [1:0]  burst_type;

  constraint c_aligned {
    addr[1:0] == 2'b00;  // 4字节对齐
  }

  constraint c_burst {
    burst_type inside {0, 1, 2};  // 不支持保留类型3
  }

  function new(string name = "my_transaction");
    super.new(name);
  endfunction
endclass
```

### 7.2 在Sequence中使用随机化

```verilog
class my_sequence extends uvm_sequence #(my_transaction);
  `uvm_object_utils(my_sequence)

  rand int unsigned num_txns;

  constraint c_num {
    num_txns inside {[10:50]};
  }

  task body();
    my_transaction txn;
    repeat (num_txns) begin
      txn = my_transaction::type_id::create("txn");
      start_item(txn);
      assert(txn.randomize()) else
        `uvm_fatal("SEQ", "randomize failed")
      finish_item(txn);
    end
  endtask
endclass
```

### 7.3 factory override 与约束扩展

```verilog
// 子类添加更严格的约束
class my_transaction_strict extends my_transaction;
  `uvm_object_utils(my_transaction_strict)

  constraint c_strict {
    addr inside {[32'h0000_0000:32'h0000_FFFF]};  // 仅64KB地址空间
    cmd inside {[0:15]};  // 仅支持前16条命令
  }
endclass
```

> [!info] 相关UVM机制
> - `uvm_config_db` 可用于在运行时传递约束参数，实现配置与约束联动
> - `uvm_sequence::randomize() with {}` 支持内联约束
> - `do_randomize()` 和 `uvm_object::create()` 集成工厂机制

## 8. 常见应用示例

### 8.1 生成合法的AHB传输

```verilog
class ahb_transaction extends uvm_sequence_item;
  rand bit [31:0] haddr;
  rand bit [2:0]  hsize;   // 0=byte, 1=halfword, 2=word
  rand bit [1:0]  htrans;  // IDLE, BUSY, NONSEQ, SEQ
  rand bit [31:0] hwdata;

  constraint c_ahb {
    // HSIZE不能超过总线宽度
    hsize <= 3'b010;  // 最大32位

    // 地址对齐
    if (hsize == 3'b001) haddr[0] == 1'b0;     // 半字对齐
    if (hsize == 3'b010) haddr[1:0] == 2'b00;  // 字对齐

    // 传输类型约束
    htrans inside {2'b00, 2'b10, 2'b11};  // 无BUSY
  }
endclass
```

### 8.2 随机化内存操作序列

```verilog
class mem_sequence extends uvm_sequence #(ahb_transaction);
  rand bit [31:0] base_addr;
  rand int unsigned num_ops;
  rand bit read_write_ratio;  // 0=读，1=写

  constraint c_mem {
    base_addr[1:0] == 2'b00;
    num_ops inside {[1:64]};
    read_write_ratio dist {0 :/ 3, 1 :/ 7};  // 70%写，30%读
  }

  task body();
    ahb_transaction txn;
    repeat (num_ops) begin
      txn = ahb_transaction::type_id::create("txn");
      start_item(txn);
      assert(txn.randomize() with {
        txn.haddr inside {[base_addr : base_addr + 4*num_ops]};
      }) else `uvm_fatal("SEQ", "randomize failed")
      finish_item(txn);
    end
  endtask
endclass
```

### 8.3 条件约束 — 覆盖所有模式

```verilog
class mode_test extends uvm_sequence #(my_transaction);
  randc bit [1:0] test_mode;  // randc确保4种模式都覆盖

  task body();
    my_transaction txn;
    repeat (16) begin
      txn = my_transaction::type_id::create("txn");
      start_item(txn);
      assert(txn.randomize() with {
        if (test_mode == 2'b00) txn.cmd inside {[0:3]};
        else if (test_mode == 2'b01) txn.cmd inside {[4:7]};
        else if (test_mode == 2'b10) txn.cmd inside {[8:11]};
        else txn.cmd inside {[12:15]};
      }) else `uvm_fatal("SEQ", "randomize failed")
      finish_item(txn);
    end
  endtask
endclass
```

## 9. 调试技巧

### 9.1 检查randomize()返回值

```verilog
// 最基本的调试手段：务必检查返回值
assert(obj.randomize()) else begin
  `uvm_error("RAND", $sformatf("randomize failed for %s", obj.get_name()))
end
```

### 9.2 使用 +svseed 命令行参数

```bash
# 指定随机种子复现失败场景
simv +svseed=12345

# UVM中使用
simv +UVM_VERBOSITY=UVM_HIGH +svseed=12345
```

### 9.3 逐步放松约束排查冲突


### 9.4 使用pre_randomize打印上下文

```verilog
class DebugItem;
  rand bit [7:0] val;

  function void pre_randomize();
    $display("[DEBUG] pre_randomize: val before = %0d", val);
  endfunction

  function void post_randomize();

  endfunction
```

### 9.5 使用get_randstate/set_randstate复现

```verilog
// 保存随机状态
string rand_state = item.get_randstate();

// 复现完全相同的随机序列
item.set_randstate(rand_state);
item.randomize();  // 产生与之前相同的值
```

### 9.6 常见错误清单

| 错误现象 | 可能原因 | 解决方法 |
|----------|----------|----------|
| randomize返回0 | 约束空间为空 | 逐步放松约束排查 |
| 随机值分布不均 | 求解器偏差 | 使用`dist`约束或调整`solve...before` |
| randc值重复出现 | randc变量重置 | 检查是否对randc变量做了手动赋值 |
| inline约束不生效 | 与类内约束冲突 | 检查内联约束与类约束的交集 |
| 数组随机化失败 | 数组大小约束冲突 | 使用`size()`约束数组长度 |
