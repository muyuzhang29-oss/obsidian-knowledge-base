# 03-覆盖率收敛策略

> [!abstract] 概述
> 收录覆盖率分析方法、收敛策略和常见问题解决方案，帮助高效达成验证目标。

---

## 覆盖率分析方法

### 1. 覆盖率类型回顾

```
覆盖率
├── 代码覆盖率 (Code Coverage)
│   ├── 行覆盖率 (Line Coverage)
│   ├── 分支覆盖率 (Branch Coverage)
│   ├── 条件覆盖率 (Condition Coverage)
│   ├── 状态机覆盖率 (FSM Coverage)
│   ├── 路径覆盖率 (Path Coverage)
│   └── 翻转覆盖率 (Toggle Coverage)
│
└── 功能覆盖率 (Functional Coverage)
    ├── 事务覆盖率 (Transaction Coverage)
    ├── 协议覆盖率 (Protocol Coverage)
    ├── 配置覆盖率 (Configuration Coverage)
    └── 时序覆盖率 (Sequence Coverage)
```

### 2. 覆盖率收集配置

```bash
# VCS 覆盖率收集
vcs -full64 -sverilog \
    -cm line+cond+fsm+tgl+branch \
    -cm_dir coverage \
    -cm_name test_name \
    -f filelist.f

# xrun 覆盖率收集
xrun -sv -uvm \
    -coverage all \
    -covfile coverage.cfg \
    -f filelist.f

# 覆盖率配置文件 (coverage.cfg)
-select line
-select cond
-select fsm
-select toggle
-select branch
-exclude module tb_top
```

### 3. 覆盖率分析流程

```
覆盖率分析流程
├── 1. 收集初始覆盖率
│   ├── 运行基础测试
│   ├── 生成覆盖率报告
│   └── 识别覆盖空洞
├── 2. 分析覆盖空洞
│   ├── 代码覆盖率空洞
│   │   ├── 死代码
│   │   ├── 不可达路径
│   │   └── 冗余逻辑
│   └── 功能覆盖率空洞
│       ├── 未触发场景
│       ├── 边界条件
│       └── 异常路径
├── 3. 制定收敛策略
│   ├── 定向测试
│   ├── 约束调整
│   └── 覆盖点优化
└── 4. 执行与验证
    ├── 运行新测试
    ├── 合并覆盖率
    └── 验证收敛效果
```

## 收敛策略

### 1. 代码覆盖率收敛

#### 策略一：识别死代码

```verilog
// 问题：某些代码行永远无法执行
// 原因：条件永远为假，或逻辑冗余

// 示例：死代码
if (0) begin  // 永远不执行
    // ...
end

// 解决方案：
// 1. 删除死代码
// 2. 修复条件逻辑
// 3. 添加测试场景
```

#### 策略二：覆盖所有分支

```verilog
// 问题：某些分支未覆盖
// 示例：未覆盖的 else 分支
if (condition) begin
    // 分支 1：已覆盖
end else begin
    // 分支 2：未覆盖
end

// 解决方案：
// 1. 创建测试用例触发 else 分支
// 2. 调整约束条件
// 3. 添加定向测试
```

#### 策略三：状态机全覆盖

```verilog
// 问题：某些状态转换未覆盖
typedef enum logic [2:0] {
    IDLE   = 3'b000,
    START  = 3'b001,
    READ   = 3'b010,
    WRITE  = 3'b100,
    DONE   = 3'b101
} state_t;

// 覆盖目标：
// - 所有状态
// - 所有合法转换
// - 边界条件

// 解决方案：
// 1. 分析状态转换图
// 2. 创建覆盖所有转换的测试
// 3. 使用覆盖率驱动的测试生成
```

### 2. 功能覆盖率收敛

#### 策略一：细化覆盖点

```verilog
// 问题：覆盖点过于粗糙
// 示例：粗粒度覆盖点
coverpoint cmd {
    bins read  = {READ};
    bins write = {WRITE};
}

// 优化：细粒度覆盖点
coverpoint cmd {
    bins read       = {READ};
    bins write      = {WRITE};
    bins read_burst = {READ_BURST};
    bins write_burst = {WRITE_BURST};
    bins idle       = {IDLE};
}

// 进一步优化：带条件的覆盖点
coverpoint cmd {
    bins read_normal  = {READ} iff (!burst_mode);
    bins read_burst   = {READ} iff (burst_mode);
    bins write_normal = {WRITE} iff (!burst_mode);
    bins write_burst  = {WRITE} iff (burst_mode);
}
```

#### 策略二：交叉覆盖优化

```verilog
// 问题：交叉覆盖率低
covergroup my_cg;
    cmd: coverpoint trans.cmd {
        bins read  = {READ};
        bins write = {WRITE};
    }
    
    addr: coverpoint trans.addr {
        bins low  = {[0:32'hFFF]};
        bins mid  = {[32'h1000:32'hEFFF]};
        bins high = {[32'hF000:32'hFFFF]};
    }
    
    // 交叉覆盖
    cmd_x_addr: cross cmd, addr {
        // 指定需要覆盖的交叉项
        bins read_low  = binsof(cmd.read) && binsof(addr.low);
        bins read_mid  = binsof(cmd.read) && binsof(addr.mid);
        bins read_high = binsof(cmd.read) && binsof(addr.high);
        bins write_low  = binsof(cmd.write) && binsof(addr.low);
        bins write_mid  = binsof(cmd.write) && binsof(addr.mid);
        bins write_high = binsof(cmd.write) && binsof(addr.high);
    }
endgroup
```

#### 策略三：边界条件覆盖

```verilog
// 问题：边界条件未覆盖
coverpoint length {
    bins zero     = {0};
    bins single   = {1};
    bins small    = {[2:7]};
    bins medium   = {[8:31]};
    bins large    = {[32:127]};
    bins max      = {128};
    bins overflow = {[129:$]};
}

// 解决方案：
// 1. 创建边界值测试
// 2. 调整约束分布
// 3. 添加定向测试
```

### 3. 约束调整策略

#### 策略一：调整随机分布

```verilog
// 问题：某些场景难以随机触发
class transaction extends uvm_sequence_item;
    rand bit [31:0] addr;
    rand bit [2:0]  len;
    
    // 原始约束（均匀分布）
    constraint original_c {
        addr inside {[0:32'hFFFF]};
        len  inside {[0:7]};
    }
    
    // 优化约束（加权分布）
    constraint optimized_c {
        addr dist {
            [0:32'hFFF]       :/ 20,      // 低地址
            [32'h1000:32'hEFFF] :/ 60,     // 中地址
            [32'hF000:32'hFFFF] :/ 20      // 高地址
        };
        len dist {
            0 := 10,           // 零长度
            1 := 20,           // 单拍
            [2:4] := 40,       // 短突发
            [5:7] := 30        // 长突发
        };
    }
endclass
```

#### 策略二：添加定向约束

```verilog
// 问题：特定场景难以触发
class directed_seq extends uvm_sequence#(transaction);
    virtual task body();
        // 定向测试：边界条件
        req.addr = 32'h0000_0000;  // 最小地址
        req.len  = 3'b000;         // 最小长度
        start_item(req);
        finish_item(req);
        
        // 定向测试：最大值
        req.addr = 32'hFFFF_FFFF;  // 最大地址
        req.len  = 3'b111;         // 最大长度
        start_item(req);
        finish_item(req);
        
        // 定向测试：特殊模式
        req.addr = 32'h1234_5678;
        req.len  = 3'b101;
        req.burst_type = WRAP;     // 回卷突发
        start_item(req);
        finish_item(req);
    endtask
endclass
```

#### 策略三：约束排除

```verilog
// 问题：非法组合干扰覆盖率
class transaction extends uvm_sequence_item;
    rand bit [1:0] mode;
    rand bit [31:0] addr;
    
    // 排除非法组合
    constraint legal_c {
        // 模式 0 只能访问低地址
        if (mode == 2'b00)
            addr inside {[0:32'h1FFF]};
        
        // 模式 1 只能访问高地址
        if (mode == 2'b01)
            addr inside {[32'hE000:32'hFFFF]};
        
        // 模式 2 和 3 可以访问所有地址
    }
endclass
```

## 常见问题及解决

### 问题 1：代码覆盖率不收敛

**现象**：某些代码行永远无法达到 100% 覆盖率。

**原因分析**：
```
代码覆盖率不收敛
├── 死代码
│   ├── 条件永远为假
│   ├── 逻辑冗余
│   └── 调试代码残留
├── 不可达路径
│   ├── 状态机死锁
│   ├── 互斥条件
│   └── 异常处理
└── 测试不足
    ├── 场景缺失
    ├── 边界未覆盖
    └── 异常未测试
```

**解决方案**：
```verilog
// 1. 识别死代码
if (0) begin  // 删除或修复
    // ...
end

// 2. 修复不可达路径
// 分析状态机，确保所有状态可达

// 3. 添加测试场景
class corner_case_test extends uvm_test;
    // 添加边界条件测试
    // 添加异常路径测试
endclass
```

### 问题 2：功能覆盖率不收敛

**现象**：某些功能覆盖点永远无法达到目标。

**原因分析**：
```
功能覆盖率不收敛
├── 覆盖点定义不当
│   ├── 粒度过粗
│   ├── 条件过严
│   └── 遗漏场景
├── 约束限制
│   ├── 约束过紧
│   ├── 分布不均
│   └── 非法组合
└── 测试策略
    ├── 测试不足
    ├── 场景单一
    └── 随机性不足
```

**解决方案**：
```verilog
// 1. 优化覆盖点定义
coverpoint cmd {
    bins read[]  = {READ};   // 分离每个值
    bins write[] = {WRITE};
    bins idle    = {IDLE};
}

// 2. 调整约束
constraint adjusted_c {
    addr dist {
        [0:32'hFFF]       :/ 30,
        [32'h1000:32'hEFFF] :/ 40,
        [32'hF000:32'hFFFF] :/ 30
    };
}

// 3. 添加定向测试
class coverage_driven_seq extends uvm_sequence#(transaction);
    virtual task body();
        // 针对未覆盖场景
        // ...
    endtask
endclass
```

### 问题 3：覆盖率数据过大

**现象**：覆盖率数据库过大，分析困难。

**原因分析**：
```
覆盖率数据过大
├── 实例过多
│   ├── 每个实例独立计数
│   └── 重复实例
├── 覆盖点过多
│   ├── 粒度过细
│   └── 冗余覆盖点
└── 交叉覆盖爆炸
    ├── 维度过高
    └── 组合过多
```

**解决方案**：
```verilog
// 1. 合并实例
covergroup my_cg;
    option.per_instance = 0;        // 不按实例计数
    option.merge_instances = 1;     // 合并所有实例
endgroup

// 2. 优化覆盖点
coverpoint cmd {
    bins read  = {READ, READ_BURST};   // 合并相似场景
    bins write = {WRITE, WRITE_BURST};
}

// 3. 限制交叉覆盖
cross cmd, addr {
    // 只覆盖关键交叉项
    bins read_low  = binsof(cmd.read) && binsof(addr.low);
    bins write_high = binsof(cmd.write) && binsof(addr.high);
    // 忽略其他组合
}
```

### 问题 4：覆盖率合并失败

**现象**：多次仿真的覆盖率数据无法合并。

**原因分析**：
```
覆盖率合并失败
├── 设计结构变化
│   ├── 模块重命名
│   ├── 信号重命名
│   └── 层次变化
├── 覆盖组定义变化
│   ├── 覆盖点变化
│   ├── bin 定义变化
│   └── 选项变化
└── 工具版本不匹配
    ├── 仿真器版本
    └── 覆盖率工具版本
```

**解决方案**：
```bash
# 1. 确保设计结构一致
# 使用相同的代码版本

# 2. 确保覆盖组定义一致
# 使用相同的测试代码

# 3. 使用兼容的工具版本
# 记录工具版本信息

# 4. 使用正确的合并命令
# VCS
urg -dir test1.vdb test2.vdb -out merged

# xrun/imc
imc -merge -out merged -input test1.ucdb test2.ucdb
```

### 问题 5：覆盖率目标难以达成

**现象**：长时间仿真后覆盖率仍不达标。

**原因分析**：
```
覆盖率目标难以达成
├── 目标设置过高
│   ├── 不切实际的目标
│   └── 遗漏不可达场景
├── 测试策略不当
│   ├── 测试不足
│   ├── 场景单一
│   └── 随机性不足
└── 设计复杂度
    ├── 状态空间过大
    ├── 条件组合爆炸
    └── 时序依赖
```

**解决方案**：
```verilog
// 1. 调整覆盖率目标
covergroup my_cg;
    option.goal = 90;  // 调整为合理目标
endgroup

// 2. 优化测试策略
class optimized_test extends uvm_test;
    // 增加测试数量
    // 多样化测试场景
    // 使用覆盖率驱动
endclass

// 3. 分阶段收敛
// 阶段 1：基础场景 80%
// 阶段 2：边界条件 90%
// 阶段 3：异常场景 95%
```
