# 01-仿真编译错误

> [!abstract] 概述
> 收录 VCS/xrun 仿真编译阶段常见错误及解决方案，帮助快速定位和修复编译问题。

---

## 常见编译错误及解决方案

### 1. 语法错误 (Syntax Error)

```
Error: syntax error near "endmodule"
Error: expecting ';'
```

**原因**: 缺少分号、括号不匹配、关键字拼写错误。

**解决方案**:
```systemverilog
// 检查常见语法问题
// 1. 分号缺失
always_ff @(posedge clk) begin
    data <= value;    // 不要漏掉分号
end

// 2. 括号匹配
if (condition) begin
    // ...
end                    // 确保 end 与 begin 配对

// 3. 关键字拼写
always_ff @(posedge clk)  // 不是 always_ff
```

### 2. 模块未定义 (Module Not Found)

```
Error: Module 'xxx' not found
Error: Cannot find module 'yyy'
```

**原因**: 模块名拼写错误、文件未包含在 filelist 中、路径错误。

**解决方案**:
```bash
# 1. 检查 filelist 是否包含该模块
grep "module_name" filelist.f

# 2. 检查模块名大小写
# Verilog 区分大小写！

# 3. 检查文件路径
ls -la ../rtl/module_name.sv

# 4. 添加 include 路径
xrun -incdir ../rtl -incdir ../tb -f filelist.f
```

### 3. 信号未声明 (Undeclared Signal)

```
Error: 'signal_name' is not declared
Error: Implicit net declarations are not allowed
```

**原因**: 信号未声明、声明顺序错误、作用域问题。

**解决方案**:
```systemverilog
// 1. 显式声明所有信号
logic [31:0] data_out;
wire         valid;

// 2. 检查声明顺序（SystemVerilog 要求先声明后使用）

// 3. 检查接口信号访问
// 使用 virtual interface
virtual my_if vif;
```

### 4. 类型不匹配 (Type Mismatch)

```
Error: Cannot assign logic to wire
Error: Type mismatch in assignment
```

**原因**: 类型不兼容、位宽不匹配。

**解决方案**:
```systemverilog
// 1. 类型匹配
logic [31:0] data_reg;
wire  [31:0] data_wire;
assign data_wire = data_reg;  // OK

// 2. 位宽匹配
logic [7:0]  byte_data;
logic [31:0] word_data;
assign word_data = {24'b0, byte_data};  // 显式扩展

// 3. 使用类型转换
int  int_val;
logic [31:0] logic_val;
assign logic_val = logic'(int_val);
```

### 5. 时序控制错误

```
Error: Cannot use delays in synthesizable code
Error: Non-synthesizable construct
```

**原因**: 在 RTL 代码中使用了仿真专用构造。

**解决方案**:
```systemverilog
// RTL 代码中避免使用：
// #delay
// initial 块
// $display, $finish
// fork/join (在综合代码中)

// 使用 `ifdef 隔离仿真代码
`ifdef SIMULATION
    initial begin
        $display("Simulation started");
    end
`endif
```

### 6. UVM 相关编译错误

```
Error: Class 'uvm_component' not found
Error: UVM library not found
```

**原因**: UVM 库路径错误、版本不匹配。

**解决方案**:
```bash
# 1. 指定正确的 UVM 路径
xrun -sv -uvmhome $UVM_HOME -f filelist.f

# 2. 检查 UVM 版本
xrun -sv -uvmversion 1.2 -f filelist.f

# 3. 使用预编译 UVM
xrun -sv -uvm -f filelist.f
```

### 7. 文件编码问题

```
Error: Illegal character in source file
Error: UTF-8 BOM detected
```

**原因**: 文件编码不正确、包含非法字符。

**解决方案**:
```bash
# 1. 转换文件编码
iconv -f GBK -t UTF-8 file.sv > file_utf8.sv

# 2. 检查文件编码
file --mime-encoding file.sv

# 3. 删除 BOM
sed -i '1s/^\xEF\xBB\xBF//' file.sv
```

### 8. 宏定义问题

```
Error: Macro 'MACRO_NAME' is not defined
Error: Recursive macro definition
```

**原因**: 宏未定义、递归定义、作用域问题。

**解决方案**:
```systemverilog
// 1. 检查宏定义
`define MY_MACRO 1
`ifdef MY_MACRO
    // ...
`endif

// 2. 避免递归定义
`define A `B  // 可能导致递归

// 3. 使用参数化宏
`define ADDR_WIDTH 32
logic [`ADDR_WIDTH-1:0] addr;
```

## VCS 编译选项

### 基本编译选项

| 选项 | 说明 |
|------|------|
| `-full64` | 64 位模式 |
| `-sverilog` | 启用 SystemVerilog |
| `-ntb_opts uvm` | 启用 UVM |
| `-timescale=1ns/1ps` | 时间刻度 |
| `-debug_access+all` | 调试访问 |
| `-kdb` | 生成 KDB 数据库 |
| `-l <logfile>` | 日志文件 |

### 常用编译命令

```bash
# 基本编译
vcs -full64 -sverilog -timescale=1ns/1ps \
    -f filelist.f \
    -l compile.log

# UVM 编译
vcs -full64 -sverilog \
    -ntb_opts uvm \
    -timescale=1ns/1ps \
    -f filelist.f \
    -l compile.log

# 调试模式编译
vcs -full64 -sverilog \
    -debug_access+all \
    -kdb \
    -f filelist.f \
    -l compile.log

# 覆盖率编译
vcs -full64 -sverilog \
    -cm line+cond+fsm+tgl+branch \
    -cm_dir coverage \
    -f filelist.f \
    -l compile.log
```

### 编译警告处理

```bash
# 抑制特定警告
vcs -full64 -sverilog \
    -warning=noSVA-DEF \
    -warning=noIDTS \
    -f filelist.f

# 将警告视为错误
vcs -full64 -sverilog \
    -warning error=SVA-DEF \
    -f filelist.f

# 显示所有警告
vcs -full64 -sverilog \
    -warning all \
    -f filelist.f
```

## xrun 编译选项

### 基本编译选项

| 选项 | 说明 |
|------|------|
| `-sv` | 启用 SystemVerilog |
| `-uvm` | 启用 UVM |
| `-uvmhome <path>` | UVM 库路径 |
| `-timescale <ts>` | 时间刻度 |
| `-compile` | 仅编译 |
| `-f <file>` | 读取 filelist |
| `-incdir <dir>` | 包含目录 |

### 常用编译命令

```bash
# 基本编译
xrun -sv -f filelist.f -compile

# UVM 编译
xrun -sv -uvm -f filelist.f -compile

# 调试模式编译
xrun -sv -uvm -access +rwc -f filelist.f -compile

# 覆盖率编译
xrun -sv -uvm -coverage all -f filelist.f -compile
```

## 编译脚本模板

### Makefile 模板

```makefile
# ============== 配置 ==============
PROJECT    = my_project
TOP_MODULE = tb_top
SIMULATOR  = xrun  # 或 vcs

# 源文件
RTL_DIR    = ../rtl
TB_DIR     = ../tb
FILELIST   = filelist.f

# 编译选项
COMPILE_OPTS = -sv -uvm
TIMESCALE    = 1ns/1ps
LOG_DIR      = logs

# ============== 目标 ==============
.PHONY: compile clean

compile:
	@mkdir -p $(LOG_DIR)
ifeq ($(SIMULATOR),xrun)
	xrun $(COMPILE_OPTS) \
		-timescale $(TIMESCALE) \
		-f $(FILELIST) \
		-l $(LOG_DIR)/compile.log \
		-compile
else
	vcs -full64 $(COMPILE_OPTS) \
		-timescale=$(TIMESCALE) \
		-f $(FILELIST) \
		-l $(LOG_DIR)/compile.log
endif

clean:
	rm -rf $(LOG_DIR) INCA_libs csrc *.log
```

### Shell 脚本模板

```bash
#!/bin/bash
# compile.sh - 编译脚本

set -e

# 配置
PROJECT="my_project"
SIMULATOR="xrun"
FILELIST="filelist.f"
LOG_DIR="logs"

# 创建日志目录
mkdir -p $LOG_DIR

# 编译函数
compile_xrun() {
    echo "Compiling with xrun..."
    xrun -sv -uvm \
        -timescale 1ns/1ps \
        -f $FILELIST \
        -l $LOG_DIR/compile.log \
        -compile
}

compile_vcs() {
    echo "Compiling with VCS..."
    vcs -full64 -sverilog \
        -ntb_opts uvm \
        -timescale=1ns/1ps \
        -f $FILELIST \
        -l $LOG_DIR/compile.log
}

# 执行编译
case $SIMULATOR in
    xrun) compile_xrun ;;
    vcs)  compile_vcs ;;
    *)    echo "Unknown simulator: $SIMULATOR"; exit 1 ;;
esac

echo "Compilation completed. Check $LOG_DIR/compile.log for details."
```

### Filelist 模板

```bash
# filelist.f - 文件列表模板

# 包含目录
+incdir+../rtl/include
+incdir+../tb

# RTL 文件
../rtl/top.sv
../rtl/submodule.sv

# 接口文件
../rtl/interface/my_if.sv

# 测试平台文件
../tb/tb_top.sv
../tb/env.sv
../tb/test.sv

# UVM 文件（如果需要）
# $(UVM_HOME)/src/uvm.sv
```

## 编译错误排查流程

```
编译错误
├── 语法错误
│   ├── 检查分号、括号
│   ├── 检查关键字拼写
│   └── 检查 begin/end 配对
├── 模块/信号未定义
│   ├── 检查 filelist
│   ├── 检查 include 路径
│   └── 检查大小写
├── 类型错误
│   ├── 检查类型声明
│   ├── 检查位宽匹配
│   └── 使用类型转换
├── UVM 错误
│   ├── 检查 UVM 路径
│   ├── 检查 UVM 版本
│   └── 检查 UVM 宏定义
└── 其他错误
    ├── 检查文件编码
    ├── 检查宏定义
    └── 查看编译日志
```
