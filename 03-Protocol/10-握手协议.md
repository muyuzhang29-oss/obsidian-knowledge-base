# 握手协议

## 1. 握手协议基础

### 1.1 什么是握手协议

握手协议是数字设计中模块间通信的基础机制。两个模块通过交换控制信号（如 `valid`、`ready`）来协调数据传输，确保发送方和接收方在**双方都准备就绪**时才完成一次数据交接。

核心思想：**没有时钟级的精确同步，就用信号级的协商来达成共识。**

### 1.2 为什么需要握手

| 问题 | 握手如何解决 |
|------|-------------|
| **速率匹配** | 发送方和接收方工作在不同频率或处理速度不同，握手允许快的一方等待慢的一方 |
| **数据完整性** | 只有双方都确认的数据才会被采样，避免采到无效/过期数据 |
| **流量控制** | 接收方通过 `ready` 信号反压发送方，防止 FIFO 溢出或数据丢失 |
| **错误处理** | 协议可扩展为带应答（Ack/Nak），支持重传和错误恢复 |

### 1.3 常见握手协议对比

| 特性 | Valid-Ready | Req-Ack | Ack-Nak |
|------|------------|---------|---------|
| 典型应用 | AXI-Stream, APB | AXI 写地址/数据通道 | PCIe, 网络协议 |
| 信号数 | 2 (valid, ready) | 2 (req, ack) | 2+ (req, ack/nak, retry) |
| 数据方向 | 单向数据 + 反向流控 | 请求-应答 | 请求-确认/拒绝 |
| 反压支持 | 原生支持（ready 低=反压） | 需额外机制 | 通过 Nak+重传实现 |
| 复杂度 | 低 | 中 | 高 |
| 典型吞吐 | 每拍一个数据 | 每次握手一个数据 | 取决于重传率 |

## 2. Valid-Ready 握手协议

### 2.1 协议规范

Valid-Ready 是最常用的握手协议，广泛用于 AXI-Stream、APB 等总线协议。

**三条核心规则：**

```
规则1: 发送方可在任意时钟周期断言 valid，无需等待 ready
规则2: valid 断言后，必须保持稳定，直到握手完成
规则3: 数据在 valid & ready 同时为高的时钟沿被采样（握手完成）
```

**信号定义：**

| 信号 | 方向 | 说明 |
|------|------|------|
| `valid` | 发送方 -> 接收方 | 发送方表示数据/地址/控制信息有效 |
| `ready` | 接收方 -> 发送方 | 接收方表示可以接收数据 |
| `data` | 发送方 -> 接收方 | 传输的数据（宽度可配置） |

**握手完成条件：**

```verilog
// 在时钟上升沿，当 valid 和 ready 同时为高时，数据传输完成
handshake_complete = valid & ready;
```

### 2.2 时序图

```
        ___     ___     ___     ___     ___     ___     ___
clk    |   |___|   |___|   |___|   |___|   |___|   |___|   |___
       
       ________                _________________________
valid          |______________|                         |______
       
       ________________________          _______________
ready                           |________|               
       
data   XXXXXXXX== DATA_A =====XXXXXXXXXX==== DATA_B ====XXXXXX
       
                     ^                       ^
                     |                       |
                handshake               handshake
                (采样 DATA_A)            (采样 DATA_B)
```

关键时序点：
- 第 3 拍：`valid` 拉高，`ready` 为低 -> 数据未传输，`valid` 必须保持
- 第 4 拍：`ready` 拉高 -> 握手完成，DATA_A 被采样
- 第 5 拍：`valid` 拉低（发送方可选）
- 第 7 拍：`valid` 和 `ready` 同时高 -> DATA_B 被采样（背靠背传输）

### 2.3 发送方实现（状态机）

```verilog
module handshake_master #(
    parameter DATA_WIDTH = 32
)(
    input  logic                    clk,
    input  logic                    rst_n,
    // 上游数据接口
    input  logic                    data_valid_i,
    input  logic [DATA_WIDTH-1:0]   data_i,
    output logic                    data_ready_o,
    // 握手接口
    output logic                    valid,
    output logic [DATA_WIDTH-1:0]   data,
    input  logic                    ready
);

    //==========================================================
    // 状态定义
    //==========================================================
    typedef enum logic [1:0] {
        ST_IDLE     = 2'b00,
        ST_VALID    = 2'b01,
        ST_DONE     = 2'b10
    } state_t;

    state_t state, next_state;

    //==========================================================
    // 状态寄存器
    //==========================================================
    always_ff @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            state <= ST_IDLE;
        else
            state <= next_state;
    end

    //==========================================================
    // 状态转移逻辑
    //==========================================================
    always_comb begin
        next_state = state;
        case (state)
            ST_IDLE: begin
                if (data_valid_i)
                    next_state = ST_VALID;
            end
            ST_VALID: begin
                if (ready)
                    next_state = ST_DONE;
            end
            ST_DONE: begin
                next_state = ST_IDLE;
            end
            default: next_state = ST_IDLE;
        endcase
    end

    //==========================================================
    // 输出逻辑
    //==========================================================
    // valid 仅在 ST_VALID 状态断言，符合规范
    assign valid = (state == ST_VALID);

    // data 在进入 ST_VALID 时锁存，握手期间保持稳定
    always_ff @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            data <= '0;
        end else if (state == ST_IDLE && data_valid_i) begin
            data <= data_i;  // 锁存数据
        end
    end

    // 上游反压：仅在 IDLE 状态接收新数据
    assign data_ready_o = (state == ST_IDLE);

endmodule
```

### 2.4 接收方实现（含 FIFO 缓冲）

```verilog
module handshake_slave #(
    parameter DATA_WIDTH = 32,
    parameter FIFO_DEPTH = 8
)(
    input  logic                    clk,
    input  logic                    rst_n,
    // 握手接口
    input  logic                    valid,
    input  logic [DATA_WIDTH-1:0]   data,
    output logic                    ready,
    // 下游数据接口
    output logic                    data_valid_o,
    output logic [DATA_WIDTH-1:0]   data_o,
    input  logic                    data_ready_i
);

    //==========================================================
    // FIFO 实现
    //==========================================================
    logic [DATA_WIDTH-1:0] fifo_mem [FIFO_DEPTH];
    logic [$clog2(FIFO_DEPTH)-1:0] wr_ptr, rd_ptr;
    logic [$clog2(FIFO_DEPTH):0]   count;

    wire fifo_full  = (count == FIFO_DEPTH);
    wire fifo_empty = (count == 0);

    // FIFO 写入：握手完成时写入
    wire wr_en = valid & ready & !fifo_full;

    always_ff @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            wr_ptr <= '0;
        end else if (wr_en) begin
            fifo_mem[wr_ptr] <= data;
            wr_ptr <= (wr_ptr == FIFO_DEPTH-1) ? '0 : wr_ptr + 1;
        end
    end

    // FIFO 读出
    wire rd_en = data_ready_i & !fifo_empty;

    always_ff @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            rd_ptr <= '0;
        end else if (rd_en) begin
            rd_ptr <= (rd_ptr == FIFO_DEPTH-1) ? '0 : rd_ptr + 1;
        end
    end

    // FIFO 计数
    always_ff @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            count <= '0;
        else begin
            case ({wr_en, rd_en})
                2'b10:   count <= count + 1;
                2'b01:   count <= count - 1;
                default: count <= count;  // 2'b00 或 2'b11
            endcase
        end
    end

    //==========================================================
    // ready 逻辑：FIFO 未满时可以接收
    //==========================================================
    assign ready = !fifo_full;

    //==========================================================
    // 下游输出
    //==========================================================
    assign data_valid_o = !fifo_empty;
    assign data_o = fifo_mem[rd_ptr];

endmodule
```

## 3. 握手协议验证

### 3.1 基本功能验证

```verilog
class handshake_basic_test extends uvm_test;
    `uvm_component_utils(handshake_basic_test)

    function new(string name, uvm_component parent);
        super.new(name, parent);
    endfunction

    //==========================================================
    // 测试 1: 正常数据传输
    //==========================================================
    task test_basic_transfer();
        // 驱动一笔写操作
        m_driver.send_data(32'hDEAD_BEEF);
        // 检查接收方是否收到正确数据
        `uvm_info("TEST", "Basic transfer PASSED", UVM_LOW)
    endtask

    //==========================================================
    // 测试 2: valid 不依赖 ready（关键规范要求）
    //==========================================================
    task test_valid_independence();
        // 发送方断言 valid，不关心 ready
        m_driver.assert_valid(32'hCAFE_BABE);
        // 延迟若干周期后才拉高 ready
        repeat(5) @(posedge vif.clk);
        m_driver.assert_ready();

        // 检查：valid 应在 ready 拉高前就稳定
        `uvm_info("TEST", "Valid independence PASSED", UVM_LOW)
    endtask

    //==========================================================
    // 测试 3: 握手完成条件
    //==========================================================
    task test_handshake_complete();
        fork
            // 驱动端：断言 valid
            begin
                vif.valid <= 1'b1;
                vif.data  <= 32'h1234_5678;
                @(posedge vif.clk iff (vif.valid && vif.ready));
                vif.valid <= 1'b0;
            end
            // 响应端：断言 ready
            begin
                repeat(3) @(posedge vif.clk);
                vif.ready <= 1'b1;
                @(posedge vif.clk iff (vif.valid && vif.ready));
                vif.ready <= 1'b0;
            end
        join
        `uvm_info("TEST", "Handshake completion PASSED", UVM_LOW)
    endtask

endclass
```

### 3.2 时序验证

```verilog
class handshake_timing_test extends uvm_test;
    `uvm_component_utils(handshake_timing_test)

    virtual handshake_if vif;

    //==========================================================
    // 测试 1: 建立时间检查
    // valid 必须在时钟上升沿前稳定至少 Tsetup
    //==========================================================
    task test_setup_time();
        realtime t_valid, t_clk;
        realtime setup_margin;

        // 监测 valid 变化时刻
        @(posedge vif.valid);
        t_valid = $realtime;

        // 等待下一个时钟沿
        @(posedge vif.clk);
        t_clk = $realtime;

        setup_margin = t_clk - t_valid;
        `uvm_info("TIMING", $sformatf("Setup margin: %0t", setup_margin), UVM_LOW)

        // 建立时间检查（假设 1ns 要求）
        assert (setup_margin >= 1.0ns)
            `uvm_info("TIMING", "Setup time PASSED", UVM_LOW)
        else
            `uvm_error("TIMING", "Setup time VIOLATION")
    endtask

    //==========================================================
    // 测试 2: 保持时间检查
    // valid 在时钟沿后必须保持稳定至少 Thold
    //==========================================================
    task test_hold_time();
        realtime t_valid_end, t_clk;
        realtime hold_margin;

        // 等待握手完成
        @(posedge vif.clk iff (vif.valid && vif.ready));

        // valid 变化的时刻
        @(negedge vif.valid);
        t_valid_end = $realtime;

        hold_margin = t_valid_end - $realtime;
        assert (hold_margin >= 0.5ns)
            `uvm_info("TIMING", "Hold time PASSED", UVM_LOW)
        else
            `uvm_error("TIMING", "Hold time VIOLATION")
    endtask

    //==========================================================
    // 测试 3: 时钟到输出延迟
    //==========================================================
    task test_clk_to_output();
        realtime t_clk, t_output;

        @(posedge vif.clk);
        t_clk = $realtime;

        // 等待 valid 输出变化
        @(vif.valid);
        t_output = $realtime;

        `uvm_info("TIMING", $sformatf("Clock-to-output: %0t", t_output - t_clk), UVM_LOW)
    endtask

endclass
```

### 3.3 边界条件验证

```verilog
class handshake_boundary_test extends uvm_test;
    `uvm_component_utils(handshake_boundary_test)

    virtual handshake_if vif;

    //==========================================================
    // 测试 1: valid 和 ready 同时断言（背靠背最快场景）
    //==========================================================
    task test_simultaneous_assertion();
        fork
            begin
                vif.valid <= 1'b1;
                vif.data  <= 32'hAAAA_BBBB;
            end
            begin
                vif.ready <= 1'b1;
            end
        join

        @(posedge vif.clk);
        // 第一拍就应该完成握手
        assert (vif.valid && vif.ready)
            `uvm_info("BOUNDARY", "Simultaneous assertion PASSED", UVM_LOW)
        else
            `uvm_error("BOUNDARY", "Failed on simultaneous assertion")
    endtask

    //==========================================================
    // 测试 2: valid 长时间等待 ready（反压场景）
    //==========================================================
    task test_valid_waits_ready();
        vif.valid <= 1'b1;
        vif.data  <= 32'hCCCC_DDDD;

        // 等待 100 个周期才拉高 ready
        repeat(100) @(posedge vif.clk);

        vif.ready <= 1'b1;
        @(posedge vif.clk);

        // 数据应保持稳定
        assert (vif.data == 32'hCCCC_DDDD)
            `uvm_info("BOUNDARY", "Valid waits ready PASSED", UVM_LOW)
        else
            `uvm_error("BOUNDARY", "Data changed while waiting!")

        vif.valid <= 1'b0;
        vif.ready <= 1'b0;
    endtask

    //==========================================================
    // 测试 3: ready 在 valid 之前撤销（反压撤销场景）
    //==========================================================
    task test_ready_deassert_before_valid();
        vif.valid <= 1'b1;
        vif.ready <= 1'b1;
        vif.data  <= 32'hEEEE_FFFF;

        @(posedge vif.clk);
        // 握手完成，ready 先撤销
        vif.ready <= 1'b0;
        @(posedge vif.clk);

        // valid 仍应保持（如果发送方有新数据要发）
        // 或者 valid 也撤销（本轮传输结束）
        // 行为取决于设计，这里检查不会出现 X 态
        assert (!$isunknown(vif.valid))
            `uvm_info("BOUNDARY", "Ready deassert PASSED", UVM_LOW)
        else
            `uvm_error("BOUNDARY", "valid went X after ready deassert")
    endtask

endclass
```

## 4. 常见握手错误

### 4.1 错误 1: valid 依赖 ready

这是最常见的握手协议违规。`valid` 不能因为 `ready` 为低而撤销，否则会导致组合环路或协议死锁。

**错误代码：**

```verilog
// !! 错误: valid 依赖 ready !!
always_comb begin
    if (ready)
        valid = data_valid_i;  // ready 为低时 valid 被拉低
    else
        valid = 1'b0;          // 违规: valid 被 ready 控制
end
```

问题分析：
- `valid` 变成 `ready` 的组合逻辑输出
- 如果 `ready` 同时依赖 `valid`（常见于流控），形成组合环路
- 即使不形成环路，发送方撤销 `valid` 会导致接收方已准备好的数据丢失

**正确代码：**

```verilog
// 正确: valid 独立于 ready
always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n)
        valid <= 1'b0;
    else if (data_valid_i && state == ST_IDLE)
        valid <= 1'b1;         // 断言 valid
    else if (valid && ready)
        valid <= 1'b0;         // 仅在握手完成时撤销
    // valid 断言后，即使 ready=0 也保持不变
end
```

### 4.2 错误 2: 数据不稳定

`valid` 为高时，`data` 必须保持稳定。如果 `data` 在 `valid` 有效期间变化，接收方可能采样到错误数据。

**错误代码：**

```verilog
// !! 错误: data 在 valid 期间变化 !!
always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
        valid <= 1'b0;
        data  <= '0;
    end else begin
        valid <= data_valid_i;
        data  <= data_i;  // 每拍都更新，即使 valid 已断言且 ready 未响应
    end
end
```

问题分析：
- 假设第 1 拍 `valid=1, ready=0, data=0xAA`
- 第 2 拍上游 `data_i` 变为 `0xBB`，但 `ready` 仍为 0
- 第 2 拍 `data` 变为 `0xBB`，但 `valid` 仍为 1
- 接收方在 `ready` 拉高时采到 `0xBB`，而非期望的 `0xAA`

**正确代码：**

```verilog
// 正确: data 在 valid 期间锁定
logic data_latched;

always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
        valid       <= 1'b0;
        data        <= '0;
        data_latched <= 1'b0;
    end else begin
        if (data_valid_i && !data_latched) begin
            valid       <= 1'b1;
            data        <= data_i;    // 仅在首次接收时锁存
            data_latched <= 1'b1;
        end else if (valid && ready) begin
            valid       <= 1'b0;
            data_latched <= 1'b0;    // 握手完成后释放
        end
        // data 在 valid 期间保持不变
    end
end
```

### 4.3 错误 3: 死锁

当发送方等待 `ready` 才拉高 `valid`，而接收方等待 `valid` 才拉高 `ready` 时，双方互相等待，形成死锁。

**错误代码（发送方）：**

```verilog
// !! 错误: 等待 ready 后才断言 valid !!
always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n)
        valid <= 1'b0;
    else if (data_valid_i && ready)  // 必须 ready=1 才发 valid
        valid <= 1'b1;
    else
        valid <= 1'b0;
end
```

**错误代码（接收方）：**

```verilog
// !! 错误: 等待 valid 后才断言 ready !!
always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n)
        ready <= 1'b0;
    else if (valid)  // 必须 valid=1 才发 ready
        ready <= 1'b1;
    else
        ready <= 1'b0;
end
```

死锁场景：
```
发送方: "我等 ready 拉高才发 valid"
接收方: "我等 valid 拉高才发 ready"
结果: 双方都在等对方 -> 死锁
```

**正确代码（发送方独立断言）：**

```verilog
// 正确: 发送方独立断言 valid
always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n)
        valid <= 1'b0;
    else if (data_valid_i)
        valid <= 1'b1;              // 独立断言，不等 ready
    else if (valid && ready)
        valid <= 1'b0;              // 握手完成后撤销
end
```

**正确代码（接收方根据 FIFO 状态决策）：**

```verilog
// 正确: 接收方根据自身状态决定 ready
assign ready = !fifo_full;  // FIFO 未满就可以接收，不依赖 valid
```

## 5. 握手协议验证策略

### 5.1 验证计划

| 测试场景 | 覆盖目标 | 优先级 |
|----------|---------|--------|
| 正常背靠背传输 | 吞吐量、功能正确性 | P0 |
| valid 先于 ready | 协议基本行为 | P0 |
| ready 先于 valid | 协议基本行为 | P0 |
| valid/ready 同时断言 | 边界条件 | P0 |
| valid 等待 N 周期 | 反压行为 | P1 |
| ready 随机切换 | 鲁棒性 | P1 |
| FIFO 满反压 | 流控功能 | P1 |
| FIFO 空读取 | 边界条件 | P1 |
| 连续突发传输 | 吞吐量极限 | P2 |
| 随机间隔传输 | 通用场景 | P2 |

### 5.2 UVM 验证环境

```verilog
//==========================================================
// 接口定义
//==========================================================
interface handshake_if #(
    parameter DATA_WIDTH = 32
)(
    input logic clk,
    input logic rst_n
);
    logic                    valid;
    logic                    ready;
    logic [DATA_WIDTH-1:0]   data;

    // 时钟块 - Driver
    clocking drv_cb @(posedge clk);
        default input #1 output #1;
        output valid;
        output data;
        input  ready;
    endclocking

    // 时钟块 - Monitor
    clocking mon_cb @(posedge clk);
        default input #1;
        input valid;
        input ready;
        input data;
    endclocking

    // Modport
    modport DRV  (clocking drv_cb, input clk, rst_n);
    modport MON  (clocking mon_cb, input clk, rst_n);
    modport DUT  (input valid, data, clk, rst_n, output ready);

    //==========================================================
    // 断言（SVA）
    //==========================================================

    // 断言1: valid 为高时 data 不能为 X
    property p_no_x_when_valid;
        @(posedge clk) valid |-> !$isunknown(data);
    endproperty
    assert property (p_no_x_when_valid)
        else $error("Data is X/Z when valid is high");

    // 断言2: valid 不能依赖 ready（不能因 ready 拉低而撤销 valid）
    property p_valid_no_depend_ready;
        @(posedge clk) (valid && !ready) |=> valid;
    endproperty
    assert property (p_valid_no_depend_ready)
        else $error("valid deasserted because ready was low");

    // 断言3: 握手完成后 valid 可以撤销
    property p_valid_deassert_after_handshake;
        @(posedge clk) (valid && ready) |=> !valid;
    endproperty
    // 注意: 此属性可选，允许背靠背传输（下一拍 valid 仍高）

endinterface

//==========================================================
// Transaction
//==========================================================
class handshake_trans extends uvm_sequence_item;
    `uvm_object_utils(handshake_trans)

    rand bit [31:0] data;
    rand int unsigned delay_before_valid;  // valid 断言前的延迟
    rand int unsigned ready_delay;         // ready 响应延迟

    constraint c_reasonable {
        delay_before_valid inside {[0:10]};
        ready_delay inside {[0:20]};
    }

    function new(string name = "handshake_trans");
        super.new(name);
    endfunction

endclass

//==========================================================
// Driver
//==========================================================
class handshake_driver extends uvm_driver #(handshake_trans);
    `uvm_component_utils(handshake_driver)

    virtual handshake_if.DRV vif;

    function new(string name, uvm_component parent);
        super.new(name, parent);
    endfunction

    virtual task run_phase(uvm_phase phase);
        handshake_trans tr;
        forever begin
            seq_item_port.get_next_item(tr);
            drive_item(tr);
            seq_item_port.item_done();
        end
    endtask

    virtual task drive_item(handshake_trans tr);
        // 等待复位释放
        @(posedge vif.rst_n);

        // 随机延迟
        repeat(tr.delay_before_valid) @(vif.drv_cb);

        // 断言 valid 和 data
        vif.drv_cb.valid <= 1'b1;
        vif.drv_cb.data  <= tr.data;

        // 等待握手完成
        do begin
            @(vif.drv_cb);
        end while (!(vif.drv_cb.valid && vif.drv_cb.ready));

        // 撤销 valid
        vif.drv_cb.valid <= 1'b0;
    endtask

endclass

//==========================================================
// Monitor
//==========================================================
class handshake_monitor extends uvm_monitor;
    `uvm_component_utils(handshake_monitor)

    virtual handshake_if.MON vif;
    uvm_analysis_port #(handshake_trans) ap;

    function new(string name, uvm_component parent);
        super.new(name, parent);
        ap = new("ap", this);
    endfunction

    virtual task run_phase(uvm_phase phase);
        handshake_trans tr;
        forever begin
            @(vif.mon_cb);
            // 检测握手完成
            if (vif.mon_cb.valid && vif.mon_cb.ready) begin
                tr = handshake_trans::type_id::create("tr");
                tr.data = vif.mon_cb.data;
                ap.write(tr);
            end
        end
    endtask

endclass

//==========================================================
// Scoreboard
//==========================================================
class handshake_scoreboard extends uvm_scoreboard;
    `uvm_component_utils(handshake_scoreboard)

    uvm_analysis_imp #(handshake_trans, handshake_scoreboard) exp_imp;
    uvm_analysis_imp #(handshake_trans, handshake_scoreboard) act_imp;

    handshake_trans exp_queue[$];
    int match_count = 0;
    int mismatch_count = 0;

    function new(string name, uvm_component parent);
        super.new(name, parent);
        exp_imp = new("exp_imp", this);
        act_imp = new("act_imp", this);
    endfunction

    virtual function void write(handshake_trans tr);
        exp_queue.push_back(tr);
    endfunction

    virtual function void compare(handshake_trans actual);
        handshake_trans expected;
        if (exp_queue.size() == 0) begin
            `uvm_error("SCB", "Unexpected actual transaction")
            return;
        end
        expected = exp_queue.pop_front();
        if (actual.data == expected.data) begin
            match_count++;
        end else begin
            `uvm_error("SCB", $sformatf("Data mismatch: exp=0x%h, act=0x%h",
                       expected.data, actual.data))
            mismatch_count++;
        end
    endfunction

endclass
```

### 5.3 验证覆盖率

```verilog
class handshake_coverage extends uvm_subscriber #(handshake_trans);
    `uvm_component_utils(handshake_coverage)

    virtual handshake_if.MON vif;

    //==========================================================
    // 功能覆盖率
    //==========================================================
    covergroup cg_handshake @(posedge vif.clk);

        // 覆盖点 1: valid 和 ready 的组合
        cp_valid_ready: coverpoint {vif.valid, vif.ready} {
            bins idle           = {2'b00};
            bins valid_wait     = {2'b10};  // valid=1, ready=0 (等待)
            bins ready_wait     = {2'b01};  // valid=0, ready=1 (提前准备)
            bins handshake_done = {2'b11};  // 握手完成
        }

        // 覆盖点 2: 数据值范围
        cp_data: coverpoint vif.data iff (vif.valid && vif.ready) {
            bins zero     = {32'h0};
            bins max      = {32'hFFFF_FFFF};
            bins low      = {[32'h1:32'hFF]};
            bins mid      = {[32'h100:32'hFFFF]};
            bins high     = {[32'h1_0000:32'hFFFF_FFFE]};
        }

        // 覆盖点 3: 连续握手间隔
        cp_gap: coverpoint get_handshake_gap() {
            bins back_to_back = {0};       // 背靠背
            bins short_gap    = {[1:3]};   // 短间隔
            bins medium_gap   = {[4:10]};  // 中间隔
            bins long_gap     = {[11:$]};  // 长间隔
        }

        // 覆盖点 4: 反压持续时间
        cp_backpressure: coverpoint get_backpressure_cycles() {
            bins none     = {0};
            bins short    = {[1:5]};
            bins medium   = {[6:20]};
            bins long     = {[21:$]};
        }

        // 交叉覆盖
        cx_valid_ready_data: cross cp_valid_ready, cp_data;

    endgroup

    //==========================================================
    // 辅助方法
    //==========================================================
    int handshake_count = 0;
    int last_handshake_time = 0;
    int current_gap = 0;

    function int get_handshake_gap();
        return current_gap;
    endfunction

    int backpressure_start = 0;
    int backpressure_cycles = 0;

    function int get_backpressure_cycles();
        return backpressure_cycles;
    endfunction

    //==========================================================
    // 采样逻辑
    //==========================================================
    virtual function void write(handshake_trans tr);
        // 在 monitor 回调中更新覆盖统计
    endfunction

    function new(string name, uvm_component parent);
        super.new(name, parent);
        cg_handshake = new();
    endfunction

endclass

//==========================================================
// 协议覆盖率：检查所有错误场景是否被测试
//==========================================================
covergroup cg_protocol_errors @(posedge clk);

    // 是否覆盖了 valid 在 ready 低时保持的场景
    cp_valid_hold_no_ready: coverpoint {valid, ready} {
        bins valid_waits = (2'b10 => 2'b10[*2:$]);  // valid 连续等待
    }

    // 是否覆盖了背靠背传输
    cp_back_to_back: coverpoint {valid, ready} {
        bins consecutive = (2'b11 => 2'b11);  // 连续握手
    }

    // 是否覆盖了 ready 抖动（ready 拉高又拉低）
    cp_ready_toggle: coverpoint {valid, ready} {
        bins ready_glitch = (2'b11 => 2'b10 => 2'b11);
    }

endgroup
```

> **验证工程师备忘录**
> 
> 1. **永远不要让 valid 依赖 ready** -- 这是验证中最常抓到的设计 bug
> 2. **valid 为高时 data 必须稳定** -- 在 monitor 中用断言检查
> 3. **ready 可以自由切换** -- 接收方有权在任意时刻反压
> 4. **背靠背是最高效的传输模式** -- 验证覆盖率要覆盖到
> 5. **FIFO 满是最常见的反压源** -- 确保 FIFO 满时 ready 正确拉低
