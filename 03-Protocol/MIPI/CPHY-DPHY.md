# MIPI C-PHY 与 D-PHY 详解

> MIPI Alliance 制定的两种物理层标准，用于高速摄像头和显示接口

---

## 一、概述

MIPI（Mobile Industry Processor Alliance）定义了两种物理层标准：

- **D-PHY**：差分信号，独立时钟线，技术成熟
- **C-PHY**：三线编码，嵌入时钟，引脚效率高

两者都支持 CSI-2（摄像头）和 DSI（显示）协议层，主要差异在物理层。

---

## 二、D-PHY 详解

### 2.1 物理层架构

```
┌─────────────────────────────────────────────┐
│                 D-PHY Lane                   │
│                                              │
│   ┌─────────┐    ┌─────────┐                │
│   │  Data   │    │  Clock  │                │
│   │  Lane   │    │  Lane   │                │
│   │ D+ / D- │    │CLK+/CLK-│                │
│   └─────────┘    └─────────┘                │
│                                              │
│   每个 Data Lane = 1对差分线 (2根)            │
│   Clock Lane = 1对差分线 (2根)               │
└─────────────────────────────────────────────┘
```

**引脚数：** 2×N（数据）+ 2（时钟）= 2N+2

### 2.2 两种工作模式

#### LP 模式（Low-Power）

| 参数 | 值 |
|------|-----|
| 电压范围 | 0V ~ 1.2V |
| 最高速率 | 10 Mbps |
| 信号类型 | 单端信号 |
| 用途 | 控制命令、状态检测 |

#### HS 模式（High-Speed）

| 参数 | 值 |
|------|-----|
| 电压范围 | 100mV ~ 300mV（差分） |
| 最高速率 | 2.5 Gbps/lane（v2.1 最高 4.5 Gbps） |
| 信号类型 | 差分信号 |
| 终端电阻 | 100Ω |
| 用途 | 高速数据传输 |

### 2.3 LP 状态编码

| 状态 | Dp | Dn | 含义 |
|------|-----|-----|------|
| LP-11 | 1 | 1 | 空闲/停止 |
| LP-10 | 1 | 0 | HS-Rqst（请求进入HS） |
| LP-01 | 0 | 1 | Bridge |
| LP-00 | 0 | 0 | Escape Mode |

### 2.4 LP → HS 切换时序

```
LP 状态:    LP-1 (1.2V)
           ↓
           LP-0 (0V)       ← Dp 翻转到 0 (LP-01)
           ↓
           HS-0            ← 差分切换开始
           ↓
           HS 数据传输      ← 差分数据流
```

### 2.5 关键时序参数

| 参数 | 说明 | 典型值 |
|------|------|--------|
| T_HS-SETUP | HS 准备时间 | ≥ 85ns |
| T_HS-PREPARE | HS 预备时间 | 40ns + 4UI |
| T_HS-ZERO | HS 零位时间 | 105ns + 6UI |
| T_HS-TRAIL | HS 尾部时间 | 60ns + 4UI |
| T_HS-EXIT | HS 退出时间 | ≥ 100ns |
| T_WAKEUP | 唤醒时间 | ≥ 1ms |
| T_LP-X | LP 传输时间 | ≥ 50ns |
| T_CLK-PREPARE | 时钟预备时间 | 38~95ns |

## 三、C-PHY 详解

### 3.1 物理层架构

```
┌─────────────────────────────────────────────┐
│                 C-PHY Lane                   │
│                                              │
│   ┌─────────────────────────┐               │
│   │        Trio             │               │
│   │    A    B    C          │               │
│   │    │    │    │          │               │
│   └────┴────┴────┴──────────┘               │
│                                              │
│   每个 Lane = 3根线 (无独立时钟)              │
└─────────────────────────────────────────────┘
```

**引脚数：** 3×N（无独立时钟线）

### 3.2 三线编码原理

3根线（A、B、C）产生 3 对差分信号：
- (A-B)
- (B-C)
- (C-A)

每对信号有正、负两种极性，共产生 **7 种有效符号**：

| 符号 | 含义 |
|------|------|
| +x | A>B, B=C |
| -x | A<B, B=C |
| +y | B>C, C=A |
| -y | B<C, C=A |
| +z | C>A, A=B |
| -z | C<A, A=B |
| 保持 | 状态不变 |

### 3.3 编码效率

- 采用 **16b7b 编码**：每 16 位数据映射为 7 个符号
- 每符号携带 **log₂(7) ≈ 2.807 bit** 信息
- 单 Lane 最高速率：**2.5 Gsym/s ≈ 5.7 Gbps**

### 3.4 时钟嵌入

C-PHY 无需独立时钟线，时钟信息通过符号切换嵌入在数据中：
- 接收端从符号切换中恢复时钟
- 减少引脚数，降低 EMI

## 四、D-PHY vs C-PHY 对比

### 4.1 核心参数对比

| 特性 | D-PHY | C-PHY |
|------|-------|-------|
| **信号线组成** | 差分对 (2根) + 时钟线 | 三线一组 (3根) |
| **编码方式** | 差分信号 (NRZ) | 三相符号编码 |
| **时钟** | 独立差分时钟对 | 嵌入式时钟 |
| **引脚数** | 2N+2 | 3N |
| **单Lane速率** | ~2.5 Gbps | ~5.7 Gbps |
| **引脚效率** | 较低 | **高** |
| **功耗** | 相对较高 | 同等带宽下更低 |
| **复杂度** | 较简单 | 编解码复杂 |
| **EMI** | 较高（有时钟线） | **较低** |

### 4.2 引脚数对比（相同带宽下）

| 需求 | D-PHY | C-PHY |
|------|-------|-------|
| 10 Gbps | 4 Data + 1 Clock = 10 pins | 2 Trio = 6 pins |
| 20 Gbps | 8 Data + 1 Clock = 18 pins | 4 Trio = 12 pins |

### 4.3 选择建议

```
带宽需求低 + 成本敏感 + 兼容性优先 → D-PHY
引脚受限 + 高带宽需求 + 低功耗优先 → C-PHY
两者都支持 → Combo-PHY（最灵活）
```

## 五、协议层（CSI-2 / DSI）

### 5.1 CSI-2（Camera Serial Interface 2）

**用途：** 摄像头模组 → 处理器

**特点：**
- 单向传输（Camera → Host）
- 支持多种图像格式：RAW8/10/12/14, YUV, RGB
- 支持虚拟通道（VC 0~3）
- 短包 + 长包格式

### 5.2 DSI（Display Serial Interface）

**用途：** 处理器 → 显示屏

**特点：**
- 支持命令模式和视频模式
- 可双向传输（读取显示寄存器需要反向通道）
- 支持多种显示格式

### 5.3 CSI-2 数据包格式

#### 短包（Short Packet）

```
┌──────────────┬──────────────┬──────────────┐
│      DI      │   Data Field │     ECC      │
│   (1 Byte)   │   (2 Bytes)  │   (1 Byte)   │
└──────────────┴──────────────┴──────────────┘
```

- 用于帧同步和行同步控制信号
- 固定 4 字节

**常见短包类型：**

| DT 值 | 含义 |
|--------|------|
| 0x00 | Frame Start (FS) |
| 0x01 | Frame End (FE) |
| 0x02 | Line Start (LS) |
| 0x03 | Line End (LE) |

#### 长包（Long Packet）

```
┌──────────────┬──────────────┬──────────────┬──────────────┬──────────────┐
│      DI      │      WC      │     ECC      │    Payload   │   Checksum   │
│   (1 Byte)   │   (2 Bytes)  │   (1 Byte)   │  (0~65535 B) │   (2 Bytes)  │
└──────────────┴──────────────┴──────────────┴──────────────┴──────────────┘
```

- 用于传输实际图像数据
- 最长 65541 字节

**Data Type (DT) 定义：**

| DT 值 | 含义 |
|--------|------|
| 0x2A | RAW8 |
| 0x2B | RAW10 |
| 0x2C | RAW12 |
| 0x2D | RAW14 |
| 0x24 | RGB888 |
| 0x1E | YUV422 8-bit |
| 0x1F | YUV422 10-bit |

### 5.4 Virtual Channel

- 支持最多 4 个虚拟通道（VC 0~3）
- 允许多路摄像头数据通过同一物理通道传输
- 例如：前置 + 后置摄像头同时传输

## 六、应用场景

### 6.1 D-PHY 应用

| 应用 | 说明 |
|------|------|
| 普通摄像头模组 | 手机前/后置摄像头 |
| 中低带宽显示 | 720p/1080p LCD/OLED |
| 汽车倒车影像 | 传统汽车摄像头方案 |
| IoT/嵌入式摄像头 | 成本敏感场景 |
| 兼容性要求高 | 需要广泛生态支持 |

### 6.2 C-PHY 应用

| 应用 | 说明 |
|------|------|
| 高像素摄像头 | 48MP/64MP/108MP/200MP |
| 8K/高分辨率显示 | 引脚数受限但带宽需求大 |
| 手机旗舰芯片 | 引脚资源紧张 |
| 高速连拍/视频 | 4K@60fps / 8K视频采集 |
| 功耗敏感设备 | 同等带宽下减少引脚数 |

### 6.3 Combo-PHY

现代 SoC（如高通骁龙、联发科天玑）通常支持 **Combo-PHY**：
- 同一组引脚既可配置为 D-PHY，也可配置为 C-PHY
- 根据连接的传感器/显示模组类型自动切换
- 例如：高通骁龙 8 Gen1 支持 C-PHY/D-PHY 双模
