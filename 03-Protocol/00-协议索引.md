## 协议对比

> [!tip] 协议选择指南
> 根据项目需求选择合适的协议，以下维度供参考。

### 速度、复杂度与应用场景对比

| 协议 | 最高速率 | 引脚数 | 复杂度 | 功耗 | 典型应用场景 | 学习难度 |
|------|----------|--------|--------|------|-------------|----------|
| AXI4 | 400MHz+ (128-bit) | 多 (地址+数据+控制) | 高 | 中 | SoC 片上互联、DDR 控制器 | 高 |
| AXI4-Lite | 200MHz+ | 中 | 中 | 低 | 寄存器配置、低速外设 | 中 |
| AXI4-Stream | 400MHz+ | 中 | 中 | 中 | 数据流传输、DMA | 中 |
| APB | 100MHz | 少 | 低 | 低 | 外设寄存器配置 | 低 |
| I2C | 3.4MHz (Fast+) | 2 | 低 | 极低 | 传感器、EEPROM、PMIC | 低 |
| SPI | 100MHz+ | 4+ | 中 | 低 | Flash、ADC/DAC、显示屏 | 中 |
| UART | 1Mbps+ | 2 | 极低 | 极低 | 调试串口、日志输出 | 极低 |
| MIPI D-PHY | 2.5Gbps/lane | 差分对 | 高 | 中 | 摄像头 (CSI-2)、显示 (DSI) | 高 |
| MIPI C-PHY | 5.7Gbps/lane | 3 线 | 高 | 中 | 摄像头 (高密度) | 高 |
| HSMT | 数 Gbps | 差分对 | 高 | 中 | 多媒体高速传输 | 高 |

### 总线类型对比

| 特性 | AXI | APB | I2C | SPI |
|------|-----|-----|-----|-----|
| 总线宽度 | 32/64/128/256 | 8/16/32 | N/A | N/A |
| 主从关系 | 1:N | 1:N | 多主多从 | 1:N |
| 时钟 | 独立时钟 | 同步时钟 | 同步 | 同步 |
| 地址空间 | 支持 | 无 | 无 | 无 |
| 突发传输 | 支持 | 不支持 | 不支持 | 不支持 |
| 数据流控制 | READY/VALID | PSEL/PENABLE | ACK | 无 |
| 典型频率 | 400MHz+ | 100MHz | 400kHz/3.4MHz | 50MHz+ |

### 串行协议对比

| 特性 | I2C | SPI | UART |
|------|-----|-----|------|
| 线数 | 2 (SDA/SCL) | 4 (MOSI/MISO/SCK/CS) | 2 (TX/RX) |
| 主设备数 | 多主 | 单主 | 点对点 |
| 从设备数 | 多从 | 多从 | N/A |
| 数据方向 | 半双工 | 全双工 | 全双工 |
| 应答位 | 有 | 无 | 可选 |
| 速率 | 100k-5M | 1-100M+ | 9600-115200 |

### 适用场景

```
高频数据传输 -> AXI
外设配置     -> APB
多设备互联   -> I2C
高速点对点   -> SPI
串口调试     -> UART
摄像头/显示  -> MIPI
多媒体传输   -> HSMT
```

### 协议选择决策树

```
需要传输数据？
├── 是 → 数据量大？
│   ├── 是 → 片内互联？──→ AXI4
│   ├── 是 → 流式数据？──→ AXI4-Stream
│   ├── 是 → 高速串行？
│   │   ├── 摄像头/显示 ──→ MIPI (CSI-2/DSI)
│   │   └── 多媒体传输 ──→ HSMT
│   └── 否 → 外设配置？──→ APB / AXI4-Lite
└── 否 → 需要低引脚数？
    ├── 是 → 低速传感器 ──→ I2C
    ├── 是 → 高速外设 ──→ SPI
    └── 否 → 调试串口 ──→ UART
```

## 学习顺序建议

> [!tip] 推荐协议学习路径
> 从简单到复杂，逐步建立协议验证的知识体系。

```
入门阶段 (1-2 周)
├── UART ──→ 理解异步通信基础，最简单的协议
└── APB  ──→ 理解总线握手，为 AXI 打基础

基础阶段 (2-3 周)
├── I2C  ──→ 掌握多主多从、应答机制
└── SPI  ──→ 掌握全双工、片选管理

进阶阶段 (3-4 周)
├── AXI4 ──→ 掌握突发传输、乱序、Outstanding
└── AXI4-Lite / AXI4-Stream ──→ AXI 协议族扩展

高级阶段 (按需)
├── MIPI D-PHY / C-PHY ──→ 高速差分信号、物理层验证
└── HSMT ──→ 专用高速协议
```

> [!note] 学习建议
> - 每学一个协议，配合编写对应的 UVM 验证环境
> - 优先学习项目中实际使用的协议
> - 参考 [[05-Verification/UVM-Template/00-总览|UVM 模板总览]] 快速搭建验证环境

## AXI 协议族

| 变体 | 说明 |
|------|------|
| AXI3 | 原始版本，32 位数据位宽 |
| AXI4 | 扩展突发长度，支持 4KB 边界 |
| AXI4-Lite | 单次传输，无突发 |
| AXI4-Stream | 仅数据流，无地址 |

## 验证策略

### 协议验证要点

1. **时序验证**
   - 建立/保持时间
   - 响应延迟
   - 超时检测

2. **协议合规性**
   - 信号握手顺序
   - 状态机转换
   - 错误处理

3. **边界条件**
   - 突发边界
   - 空闲周期
   - 错误注入

### 常见问题

- 总线竞争
- 死锁
- 数据错位
- 时钟同步问题
