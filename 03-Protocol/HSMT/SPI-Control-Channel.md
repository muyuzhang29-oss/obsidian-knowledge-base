# HSMT SPI 控制通道完整技术手册

> 基于 QC/T 1217-2024 附录 A.3 媒体封装层 / A.5 寄存器

## 一、协议栈总览

```
┌──────────────────────────────────────────────────────────────┐
│                        应用层                                │
│    控制 Master (SoC) ←──HSMT──→ 控制 Slave (远端设备)         │
├──────────────────────────────────────────────────────────────┤
│                     媒体封装层 (A.3)                          │
│    控制业务封装包 (B类, 业务类型=0x2)                          │
│    ├── 控制读写 (子业务类型=01, 操作码 000/001)               │
│    └── 状态确认 (子业务类型=10, 状态码 000~100)               │
├──────────────────────────────────────────────────────────────┤
│                       物理层 (A.2)                            │
│    64B/66B → PLDB → RS-FEC → 加扰 → 符号映射 → 线束          │
└──────────────────────────────────────────────────────────────┘
```

## 二、媒体封装包基础 (A.3.1)

### 2.1 三种包类型

| 类型 | 用途 | 业务类型 | CRC | 链路序列号 |
|------|------|---------|-----|-----------|
| **A类** | 图像业务、信息安全 | 0x0, 0xE | CRC32 | 固定为0 |
| **B类** | 音频、**控制业务**、GPIO | 0x1, **0x2**, 0x5 | **CRC16** | 0/1交替 |
| **C类** | 传输确认包 | 0xF | CRC32 | 固定为0 |

**SPI 控制业务使用 B 类媒体封装包，业务类型 = 0x2。**

### 2.2 公共头字段

```
字节0:
  [7]   封装协议指示 (固定=1)
  [6]   链路序列号 (B类: 每发一个包取反, 初始0)
  [5]   媒体接入控制指示 (1=有MAC字段, 0=无)
  [4:0] 预留
```

### 2.3 媒体接入控制字段 (MAC)

当 `媒体接入控制指示=1` 时，包含 MAC 字段（控制业务必须=1）：

| 子字段 | 比特 | 描述 |
|--------|------|------|
| **源地址** | [5:0] | 0x00~0x1F 单播地址 |
| **目的地址** | [11:6] | 0x3F 为广播地址 |
| **QoS** | [14:12] | 服务优先级，越小越高 |
| 预留 | [15] | - |

### 2.4 B类封装包格式 (图 A.38)

```
字节0: [预留(1)] [业务类型(4)] [MAC指示(1)] [链路序列号(1)] [封装协议指示(1)]
字节1: [MAC字段 - 源地址/目的地址/QoS] (2字节)
字节2: [子业务信息] [预留] [子业务类型(2)]
字节3: [子业务信息]
字节4~N: 净荷数据
CRC16 (2字节)
```

## 三、控制业务封装 (A.3.4)

### 3.1 控制业务总格式 (图 A.66)

```
字节0: [源地址] [MAC指示(1)] [链路序列号] [封装协议指示(1)]
字节1: [QoS] [目的地址] [源地址(低2)]
字节2: [预留] [业务类型=0x2] [预留] [QoS(低2)]
字节3: [源端口] [其他子业务信息] [预留] [子业务类型]
字节4: [目的端口] [源端口(低bit)]
字节5~N: 净荷数据
CRC16
```

### 3.2 端口定义 (表 A.47)

| 端口值 | 定义 | 说明 |
|--------|------|------|
| 0x00 | 标准寄存器 | 仅用于目的端口 |
| 0x01 | 自定义寄存器 | 仅用于目的端口 |
| 0x02~0x05 | FC_0 ~ FC_3 | FC 接口 |
| **0x06** | **SPI_0** | SPI 接口 0 |
| **0x07** | **SPI_1** | SPI 接口 1 |
| **0x08** | **SPI_2** | SPI 接口 2 |
| 0x09~0x0C | UART_0 ~ UART_3 | UART 接口 |
| 0x0D~0x14 | GPIO_0 ~ GPIO_7 | GPIO 接口 |
| 0x15~0x1F | 预留 | - |

**地址二元组**（源地址，目的地址）+ **端口二元组**（源端口，目的端口）唯一确定一条控制业务传输路径。

### 3.3 子业务类型 (表 A.48)

| 子业务类型 | 含义 | 适用接口 |
|-----------|------|---------|
| **00** | I2C 透传 | 仅 I2C |
| **01** | **控制读写** | **I2C / SPI / UART** |
| **10** | **状态确认** | **I2C / SPI / UART** |
| 11 | 预留 | - |

**SPI 控制业务仅使用非透传模式（子业务类型 01 和 10）。**

> 注：控制读写和状态确认共用一个链路序列号，其他子业务类型使用独立的链路序列。

## 四、控制写操作 (操作码 000)

### 4.1 I2C/SPI 端口控制写封装包 (图 A.72)

```
字节0: [源地址(高2)] [MAC指示(1)] [链路序列号] [封装协议指示(1)]
字节1: [QoS] [目的地址(高2)] [源地址(低2)]
字节2: [预留] [业务类型=0x2] [预留] [QoS(低2)]
字节3: [源端口] [预留] [预留] [子业务类型=01]
字节4: [目的端口] [源端口(低bit)]
字节5: [扩展长度指示(1)] [长度-低4bit(4)] [操作码=000(3)]
字节6: [预留] [长度-高7bit(7)]
字节7: [远端地址] [预留]
字节8~N: 业务数据
CRC16
```

### 4.2 字段详解

| 字段 | 位置 | 说明 |
|------|------|------|
| **操作码** | 字节5[2:0] | `000` = 控制写 |
| **扩展长度指示** | 字节5[7] | 0→长度4bit(≤15字节)；1→长度11bit |
| **长度** | 字节5[6:3] + 字节6[6:0] | 长度字段与CRC之间的字节数 |
| **远端地址** | 字节7 | 远端 SPI 从节点地址 |
| **业务数据** | 字节8~N | 要写入的数据 |

### 4.3 操作码定义 (表 A.50)

| 操作码 | 含义 |
|--------|------|
| **000** | **控制写** |
| **001** | **控制读** |
| 010~101 | 预留 |
| 110 | UART 数据流写 |
| 111 | 预留 |

## 五、控制读操作 (操作码 001) — SPI 专用

### 5.1 SPI 端口控制读封装包 (图 A.77)

```
字节0: [源地址(高2)] [MAC指示(1)] [链路序列号] [封装协议指示(1)]
字节1: [QoS] [目的地址(高2)] [源地址(低2)]
字节2: [预留] [业务类型=0x2] [预留] [QoS(低2)]
字节3: [源端口] [预留] [预留] [子业务类型=01]
字节4: [目的端口] [源端口(低bit)]
字节5: [扩展长度指示(1)] [长度-低4bit(4)] [操作码=001(3)]
字节6: [预留] [长度-高7bit(7)]
字节7: [远端地址] [预留]
字节8: [读数据长度-低4bit(4)] [扩展读数据长度指示(1)] [写读间隔指示(1)] [时钟关闭指示(1)] [预留(1)]
字节9: [预留] [读数据长度-高7bit(7)]
字节10: [写读间隔]
字节11~N: 业务数据 (可选)
CRC16
```

### 5.2 SPI 特有字段详解

| 字段 | 位置 | 说明 |
|------|------|------|
| **时钟关闭指示** | 字节8[0] | 1=SPI时钟在写读间隔保持关闭；0=保持正常 |
| **写读间隔指示** | 字节8[1] | 1=写读间隔字段存在；0=不存在 |
| **扩展读数据长度指示** | 字节8[2] | 0→读数据长度4bit(≤15字节)；1→读数据长度11bit |
| **读数据长度** | 字节8[7:3] + 字节9[6:0] | 要读取的字节数 |
| **写读间隔** | 字节10 | 向SPI从节点写和读的间隔，以8个SPI时钟周期为单位 |
| **远端地址** | 字节7 | 远端 SPI 从节点地址 |
| **业务数据** | 字节11~N | 可选，先写再读时携带 |

### 5.3 操作流程

```
情况1: 有业务数据 (先写后读)
    ├── 发送: 控制读封装包 (含业务数据)
    ├── 远端: 先将业务数据写入SPI从节点
    ├── 远端: 等待写读间隔 (时钟关闭指示决定SCK行为)
    ├── 远端: 从SPI从节点读取数据
    └── 返回: 状态确认包 (状态码=010, 带读数据)

情况2: 无业务数据 (直接读)
    ├── 发送: 控制读封装包 (无业务数据)
    ├── 远端: 直接从SPI从节点读取数据
    └── 返回: 状态确认包 (状态码=010, 带读数据)

时钟关闭指示=0 时:
    └── 应忽略远端SPI接口在写读间隙时间内收到的数据
```

### 5.4 SPI 控制读 vs I2C 控制读 对比

| 字段 | SPI (图 A.77) | I2C (图 A.76) |
|------|--------------|--------------|
| 时钟关闭指示 | **有** | 无 |
| 写读间隔指示 | 有 | 有 |
| 扩展读数据长度指示 | 有 | 有 |
| 读数据长度 | 有 | 有 |
| 写读间隔 | 以8个SPI时钟周期为单位 | 以8个I2C比特时间为单位 |
| 远端地址 | SPI从节点地址 | I2C从节点地址 |

## 六、状态确认 (子业务类型 10)

### 6.1 状态码定义 (表 A.51)

| 状态码 | 含义 | 说明 |
|--------|------|------|
| **000** | **写成功** | 无额外数据 |
| **001** | **写失败** | 带8字节自定义写失败状态 |
| **010** | **读成功-带读数据** | 带业务数据 |
| **011** | **读失败-不带读数据** | 带8字节自定义读失败状态 |
| **100** | **读失败-带读数据** | 带8字节失败状态 + 业务数据 |
| 101~111 | 预留 | - |

### 6.2 写成功 (状态码=000, 图 A.80)

```
字节0~4: 通用头 (同控制写)
字节5:   [扩展长度指示(=0)] [长度] [状态码=000]
CRC16
```

### 6.3 写失败 (状态码=001, 图 A.81)

```
字节0~4: 通用头
字节5:   [长度] [扩展长度指示(=0)] [状态码=001]
字节6~13: 用户自定义写失败状态 (固定8字节)
CRC16
```

### 6.4 读成功-带读数据 (状态码=010, 图 A.82)

```
字节0~4: 通用头

字节6:   [预留] [长度-高7bit]
字节7~N: 业务数据 (读取的数据)
CRC16
```

### 6.5 读失败-不带读数据 (状态码=011, 图 A.83)

```
字节0~4: 通用头
字节5:   [长度] [扩展长度指示(=0)] [状态码=011]
字节6~13: 自定义读失败状态 (固定8字节)
CRC16
```

### 6.6 读失败-带读数据 (状态码=100, 图 A.84)

```
字节0~4: 通用头
字节5:   [扩展长度指示] [长度-低4bit] [状态码=100]
字节6:   [预留] [长度-高7bit]
字节7~14: 自定义读失败状态 (固定8字节)
字节15~N: 业务数据
CRC16
```

## 十、错误处理与重传

### 10.1 CRC 校验

- 控制业务使用 **CRC16** 校验
- 校验范围：从公共头到 CRC 之前的所有字节
- CRC 校验失败 → 丢弃该包

### 10.2 超时机制

Testbench 中的超时参数对应协议中的等待机制：

| 参数 | 含义 | 协议对应 |
|------|------|---------|
| `spis_state_ack_tout_step` | 超时计数步长 | 状态确认等待超时 |
| `spis_state_ack_tout_thrs` | 超时计数阈值 | 状态确认等待超时 |
