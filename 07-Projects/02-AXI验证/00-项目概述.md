# AXI 验证项目概述

> [!abstract] 项目简介
> 本项目通过搭建完整的 UVM 验证环境，对 AXI 主从设备进行全面功能验证。涵盖协议基础读写、突发传输、乱序响应、交织等核心场景，旨在将 AXI 协议知识与 UVM 验证方法学融会贯通。

---

## 项目目标

验证一个 AXI 主从设备系统，确保其在以下场景下功能正确：

| 目标维度 | 具体内容 |
|---------|---------|
| 协议合规 | 严格遵循 AMBA AXI4 协议规范，所有信号时序正确 |
| 功能覆盖 | 覆盖所有突发类型、响应码、对齐方式、ID 机制 |
| 边界鲁棒 | 地址边界、最大突发长度、同时读写等极端场景 |
| 性能验证 | 乱序完成、数据交织、通道并行等性能特性 |

---

## 学习目标

完成本项目后，应掌握以下能力：

1. **AXI 协议验证** -- 深入理解五通道握手机制、信号时序约束
2. **突发传输验证** -- 验证 INCR / WRAP / FIXED 三种突发类型的正确性
3. **乱序处理验证** -- 验证多 ID 事务的乱序完成与数据交织
4. **UVM 环境搭建** -- 从零构建完整的 Agent / Env / Test 层次
5. **覆盖率驱动** -- 功能覆盖率建模与定向测试闭环
