# 仿真环境脚本（Makefile + wave.tcl）

tags: #Makefile #TCL #仿真 #Xcelium

## Makefile — 编译运行脚本

### 文件位置

`code/spi_it/sim/Makefile`

### 运行逻辑

| 目标 | 命令 | 说明 |
|------|------|------|
| `make pre_comp` | 创建仿真目录 + 软链接 | 初始化环境 |
| `make compile` | `xrun -sv -f filelist_spi -top tb_spi` | 编译全部源文件 |
| `make simulate` | `xrun -r tb_spi +UVM_TESTNAME=$(TEST)` | 运行仿真 |
| `make clean` | 删除仿真目录 | 清理 |
| `make debug` | 打开 Simvision | 看波形 |

### 关键变量

```
TEST     ?= base_test    # 测试用例名
SEED     ?= random       # 随机种子
DUMP     ?= 0            # 是否 dump 波形
COV      ?= 0            # 是否收集覆盖率
SDF      ?= none         # 是否反标 SDF
MODE     ?= base_fun     # 仿真模式
TOP      = tb_spi        # 顶层模块
```

### 使用流程

```bash
# 一次完整的验证周期
make compile                              # 编译
make simulate TEST=base_test SEED=12345   # 跑测试
make debug                                # 开波形看结果
```

### 和 I2C Makefile 的区别

I2C 环境：
- 含 VIP 编译（Denali MIPI CSI-2 VIP）
- 含 SDF 反标选项
- 含 UVM 1.2 混合编译（SC 模式）
- 含覆盖率收集

SPI 环境（简化版）：
- 无 VIP，纯 RTL + TB
- 无 SDF 反标
- 无 SystemC 混合
- 无覆盖率（可选加）

## wave.tcl — 波形 Dump 脚本

### 文件位置

`code/spi_it/sim/wave.tcl`

### 运行逻辑

```tcl
# 创建波形数据库
database -open waves -shm -into waves -default

# probe 指定层级的所有信号
probe -database waves -create tb -all -depth all
```

`probe` 是 Xcelium/Simvision 的命令，控制**哪些信号进波形文件**。

### 关键参数

| 参数 | 说明 |
|------|------|
| `-database waves` | 写入名为 waves 的波形库 |
| `-create` | 创建 probe 点 |
| `-all -depth all` | 该层级下全部信号都记录 |
| `$env(TOP)` | 从环境变量读顶层模块名 |

### 如何使用

```tcl
# 方式1：手动在 Simvision 里 source
source wave.tcl

# 方式2：makefile 里传参（DUMP=1 时自动加载）
# make simulate DUMP=1
```

### 精简说明

原始脚本对 SS01_CHP0~3、subsystem_cor_inst[0~3] 各写 10+ 行重复 probe，用 `for` / `foreach` 循环替代：

```tcl
# 精简前（50行，每个 instance 单独一行）
probe -database waves -create tb.u_SS01_CHP0 -all -depth all
probe -database waves -create tb.u_SS01_CHP1 -all -depth all
...

# 精简后（2行）
for {set i 0} {$i < 4} {incr i} {
    probe -database waves -create tb.u_SS01_CHP${i} -all -depth all
}
```

需要新增 probe 点时只需在脚本末尾加一行。
