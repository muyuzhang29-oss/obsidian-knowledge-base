# 知识库美化样式使用说明

本文档介绍如何安装、使用和自定义知识库的 CSS 美化样式文件。

---

## 一、安装方法

### 1. 启用 CSS Snippet

1. 打开 Obsidian，进入 **设置** (Settings)
2. 左侧导航选择 **外观** (Appearance)
3. 找到 **CSS 代码片段** (CSS Snippets) 部分
4. 点击文件夹图标，打开 snippets 目录
5. 确认文件 `01-知识库美化.css` 已位于该目录中（或从 `07-Scripts` 复制过去）
6. 回到 Obsidian，点击刷新按钮
7. 启用 `01-知识库美化` 旁的开关

### 2. 文件位置

| 文件 | 路径 | 用途 |
|------|------|------|
| CSS 样式文件 | `07-Scripts/01-知识库美化.css` | 源文件，可编辑维护 |
| Snippet 文件 | `.obsidian/snippets/01-知识库美化.css` | Obsidian 加载的实际文件 |

> [!tip]
> 建议将 `07-Scripts/01-知识库美化.css` 作为源文件编辑，然后复制到 `.obsidian/snippets/` 目录。也可以使用符号链接让两者指向同一文件。

### 3. 符号链接方法 (推荐)

在 PowerShell 中执行以下命令，可以避免维护两份文件：

```powershell
# 以管理员身份运行 PowerShell
New-Item -ItemType SymbolicLink `
  -Path "D:\obsidian\knowledge-base\.obsidian\snippets\01-知识库美化.css" `
  -Target "D:\obsidian\knowledge-base\07-Scripts\01-知识库美化.css"
```

## 二、自定义修改方法

### 1. 修改颜色主题

在 CSS 文件顶部的 `:root` 变量区域，可以快速修改整体配色：

```css
:root {
  --kb-accent-color: #7c3aed;    /* 主题强调色 (紫色) */
  --kb-heading-color: #1e293b;   /* 标题颜色 */
  --kb-text-primary: #334155;    /* 正文颜色 */
  /* ... 更多变量 */
}
```

常用强调色推荐：

| 颜色 | Hex 值 | 风格 |
|------|--------|------|
| 紫色 (默认) | `#7c3aed` | 学术感 |
| 蓝色 | `#2563eb` | 专业感 |
| 青色 | `#0891b2` | 科技感 |
| 绿色 | `#059669` | 自然感 |
| 玫红 | `#e11d48` | 活力感 |

### 2. 修改字体

修改字体变量：

```css
:root {
  --kb-font-sans: "Inter", "Noto Sans SC", sans-serif;
  --kb-font-mono: "JetBrains Mono", "Fira Code", monospace;
  --kb-font-heading: "Inter", "Noto Sans SC", sans-serif;
}
```

> [!note]
> 确保系统中已安装对应字体，否则会回退到后续字体。

### 3. 修改间距和圆角

```css
:root {
  --kb-spacing-md: 16px;    /* 中等间距 */
  --kb-radius-md: 8px;      /* 中等圆角 */
}
```

### 4. 暗色模式

样式文件已内置暗色模式支持，会自动跟随 Obsidian 的明暗主题切换。暗色模式的变量定义在 `.theme-dark` 选择器下，可单独调整。

## 三、特殊样式使用方法

### 1. Callout 样式

在 Markdown 中使用以下语法创建 Callout：

```markdown
> [!info] 信息提示
> 这是一个信息提示框。

> [!tip] 小技巧
> 这是一个技巧提示框。

> [!warning] 警告
> 这是一个警告提示框。

> [!danger] 危险
> 这是一个危险提示框。

> [!success] 成功
> 这是一个成功提示框。

> [!example] 示例
> 这是一个示例提示框。

> [!note] 笔记
> 这是一个笔记提示框。
```

**效果预览：**

> [!info] 信息提示
> 蓝色主题，用于展示补充信息或说明。

> [!tip] 小技巧
> 绿色主题，用于分享有用的技巧和建议。

> [!warning] 警告
> 黄色主题，用于提醒需要注意的事项。

> [!danger] 危险
> 红色主题，用于标记严重问题或风险。

> [!success] 成功
> 绿色主题，用于标记完成或正确的结果。

> [!example] 示例
> 紫色主题，用于展示具体的示例代码或用法。

> [!note] 笔记
> 灰色主题，用于一般的备注信息。

### 2. 键盘快捷键

使用 `<kbd>` 标签展示键盘快捷键：

```html
按 <kbd>Ctrl</kbd> + <kbd>P</kbd> 打开命令面板
```

**效果预览：** 按 <kbd>Ctrl</kbd> + <kbd>P</kbd> 打开命令面板

### 3. 高亮文本

使用 `==文本==` 语法高亮重要内容：

```markdown
这是 ==需要重点关注== 的内容。
```

**效果预览：** 这是 ==需要重点关注== 的内容。

### 4. 标签分类

标签会根据前缀自动分类着色：

| 前缀 | 颜色 | 用途 |
|------|------|------|
| `#UVM` | 蓝色 | UVM 相关内容 |
| `#SV` / `#SystemVerilog` | 粉色 | SystemVerilog 内容 |
| `#AXI` / `#Protocol` / `#协议` | 紫色 | 协议相关内容 |
| `#Bug` / `#Issue` / `#问题` | 红色 | 问题追踪 |
| `#Important` / `#核心` / `#重要` | 橙色 | 重要内容 |
| `#Verification` / `#验证` | 青色 | 验证相关内容 |
| `#Tool` / `#工具` | 绿色 | 工具相关内容 |
| `#Script` / `#脚本` | 灰色 | 脚本相关内容 |

### 5. 任务列表

```markdown
- [x] 已完成的任务
- [ ] 待完成的任务
- [ ] 另一个待完成的任务
```

**效果预览：**
- [x] 已完成的任务（显示删除线和自定义复选框）
- [ ] 待完成的任务（紫色复选框）

### 6. Dataview 查询

样式自动适配 Dataview 插件的表格和列表视图，无需额外配置。表格会使用与普通表格一致的斑马纹和悬停效果。

## 四、效果预览

### 标题效果

- **H1**：底部紫色粗线 + 大字号
- **H2**：底部紫色细线 + 中等字号
- **H3**：左侧紫色竖线装饰
- **H4**：左侧菱形符号装饰

### 代码块效果

行内代码使用粉红色文字 + 浅灰背景，代码块使用圆角边框 + 语言标签。

### 表格效果

表头使用主题色背景，行悬停时显示淡紫色高亮，偶数行有浅灰色背景（斑马纹）。

### 链接效果

- 内部链接：紫色文字 + 虚线下划线，悬停时变为实线 + 浅紫背景
- 外部链接：蓝色文字 + 外链图标，悬停时下划线变为实线

### 分割线效果

渐变紫色线条，从透明过渡到主题色再回到透明。

## 五、常见问题

### Q: 修改 CSS 后没有生效？

1. 确认已在 设置 > 外观 > CSS 代码片段 中启用该 snippet
2. 点击 CSS 代码片段旁的刷新按钮
3. 重新打开当前笔记

### Q: 如何禁用某个效果？

在 CSS 文件中找到对应章节，注释掉或删除相关样式块。每个章节都有清晰的注释标记。

### Q: 如何添加自定义样式？

在 CSS 文件末尾追加自定义样式即可。建议使用 `/* 自定义样式 */` 注释标记，方便后续维护。

### Q: 暗色模式颜色不对？

检查 `.theme-dark` 选择器下的变量是否正确覆盖了 `:root` 中的默认值。
