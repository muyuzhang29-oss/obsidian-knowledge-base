# 01-覆盖率

## 覆盖率概述

覆盖率是衡量验证完备性的关键指标，用于回答"我们验证了多少？"的问题。

## 覆盖率类型

```
覆盖率
├── 代码覆盖率 (Code Coverage)
│   ├── 行覆盖率 (Line Coverage)
│   ├── 分支覆盖率 (Branch Coverage)
│   ├── 条件覆盖率 (Condition Coverage)
│   ├── 状态机覆盖率 (FSM Coverage)
│   ├── 路径覆盖率 (Path Coverage)
│   └── 翻转覆盖率 (Toggle Coverage)
│
└── 功能覆盖率 (Functional Coverage)
    ├── 事务覆盖率 (Transaction Coverage)
    ├── 协议覆盖率 (Protocol Coverage)
    ├── 配置覆盖率 (Configuration Coverage)
    └── 时序覆盖率 (Sequence Coverage)
```

## 代码覆盖率

### 行覆盖率 (Line Coverage)

```verilog
if (a) begin        // ✓ 已覆盖
    b = 1;          // ✓ 已覆盖
end else begin      // ✓ 已覆盖
    b = 0;          // ✓ 已覆盖
end

// 未覆盖的行
c = 1;              // ✗ 未覆盖 (死代码或不可能执行)
```

### 分支覆盖率 (Branch Coverage)

```verilog
if (condition)      // 两个分支都要测试
    statement1;     // true 分支
else
    statement2;     // false 分支

// case 语句每个分支
case (state)
    IDLE:   ...     // 分支 1
    READ:   ...     // 分支 2
    WRITE:  ...     // 分支 3
    default:...     // 分支 4
endcase
```

### 条件覆盖率 (Condition Coverage)

```verilog
// 复合条件的每个原子条件
if (a && b)   // 需要测试: a=0&b=0, a=0&b=1, a=1&b=0, a=1&b=1

if (en && (cnt < 10))
    // 4 种原子条件组合
```

### FSM 覆盖率

```verilog
typedef enum logic [2:0] {
    IDLE  = 3'b001,
    READ  = 3'b010,
    WRITE = 3'b100
} state_t;

// 覆盖所有状态转换
// IDLE -> READ, IDLE -> WRITE
// READ -> IDLE, READ -> WRITE
// WRITE -> IDLE, WRITE -> READ
```

## 功能覆盖率

### UVM 功能覆盖率实现

```verilog
// 1. 定义覆盖组
class my_coverage extends uvm_subscriber#(my_transaction);
    `uvm_component_utils(my_coverage)

    covergroup cmd_cg;
        option.per_instance = 1;
        option.name = "cmd_cg";

        // 事务类型覆盖
        cmd: coverpoint trans.cmd {
            bins read    = {READ_CMD};
            bins write   = {WRITE_CMD};
            bins idle    = {IDLE_CMD};
            bins others  = default;
        }

        // 地址覆盖
        addr: coverpoint trans.addr {
            bins low    = {[0:12'hFFF]};
            bins mid    = {[12'h1000:12'hEFFF]};
            bins high   = {[12'hF000:12'hFFFF]};
            bins others = default;
        }

        // 交叉覆盖
        cmd_x_addr: cross cmd, addr {
            bins rd_low  = binsof(cmd.read) && binsof(addr.low);
            bins wr_high = binsof(cmd.write) && binsof(addr.high);
        }
    endgroup

    function new(string name, uvm_component parent);
        super.new(name, parent);
        cmd_cg = new();
    endfunction

    function void write(T t);
        trans = t;
        cmd_cg.sample();
    endfunction
endclass
```

### 高级覆盖点定义

```verilog
covergroup config_cg with function sample(bit [1:0] mode, int len);
    option.per_instance = 1;

    mode_cp: coverpoint mode {
        bins modes[] = {[0:3]};  // 所有模式
    }

    len_cp: coverpoint len {
        bins single  = {[1:1]};
        bins burst2  = {[2:2]};
        bins burst4  = {[3:4]};
        bins burst8  = {[5:8]};
        bins others  = default;
    }

    // 加权覆盖
    option.weight = 50;

    // 覆盖率目标
    option.goal = 90;
endgroup
```

### 覆盖组选项

```verilog
covergroup my_cg;
    option.per_instance = 1;      // 每个实例独立计数
    option.name = "my_cg";        // 覆盖组名称
    option.comment = "Main covergroup";
    option.goal = 100;            // 目标覆盖率
    option.auto_bin_max = 64;     // 自动 bin 数量
    option.cross_num_print_missing = 100;  // 打印未覆盖交叉项

    // 类型覆盖（所有实例共享）
    option.type_option.merge_instances = 1;
endcovergroup
```

### 覆盖率和采样

```verilog
class my_monitor extends uvm_monitor;
    my_coverage m_cov;

    function void build_phase(uvm_phase phase);
        m_cov = my_coverage::type_id::create("m_cov", this);
    endfunction

    task run_phase(uvm_phase phase);
        forever begin
            my_transaction tr;
            @(posedge vif.clk);
            if (vif.req) begin
                tr = new();
                tr.cmd  = vif.cmd;
                tr.addr = vif.addr;
                tr.data = vif.data;
                m_cov.write(tr);  // 采样
            end
        end
    endtask
endclass
```

## 覆盖率收集

### 仿真命令

```bash
# Questa
vlog +cover=bcesfx +coverfile=cov.ocdb top.sv
vsim -coverage -cvgpevents top_tb

# VCS
urg -dir sim.vdb
dve -cov dir=sim.vdb

# Incisive
ncverilog -coverage all -entry Test -shm
```

### UVM 配置

```verilog
class my_test extends uvm_test;
    function void build_phase(uvm_phase phase);
        // 启用功能覆盖率
        uvm_config_db#(bit)::set(this, "*", "coverage_enable", 1);
        // 启用代码覆盖率
        uvm_config_db#(bit)::set(this, "*", "coverage_save", 1);
    endfunction
endclass
```

## 覆盖率收敛策略

### 1. 识别覆盖空洞

```verilog
// 发现未覆盖的组合
covergroup my_cg;
    // 分析未覆盖的条件组合
    option.cross_num_print_missing = 100;
endgroup
```

### 2. 定向测试

```verilog
// 针对覆盖空洞编写定向测试
class corner_case_seq extends uvm_sequence#(transaction);
    virtual task body();
        // 边界值
        req.addr = 0;
        req.len  = 1;
        start_item(req);
        finish_item(req);
    endtask
endclass
```

### 3. 约束调整

```verilog
class transaction extends uvm_sequence_item;
    rand bit [31:0] addr;
    rand bit [31:0] data;
    rand bit [2:0]  len;

    constraint addr_c {
        addr inside {[0:'h1000], ['hE000:'hFFFF]};  // 排除中间区域增加边界覆盖
    }

    constraint len_c {
        len dist {0:=10, [1:4]:=40, 7:=10};  // 调整分布
    }
endclass
```

## 常见问题

### 1. 覆盖率不收敛

```verilog
// 问题：某些场景难以触发
// 解决方案：降低约束、添加定向测试
constraint relaxed_c {
    addr inside {[0:'hFFFF]};  // 放宽约束
}
```

### 2. 功能覆盖率定义不当

```verilog
// 问题：覆盖点过于粗糙
// 解决方案：细化覆盖点
coverpoint addr {
    bins low    = {[0:'h0FFF]};
    bins mid    = {[10:'h7FFF]};  // 细分
    bins high   = {[10'h8000:'hFFFF]};
}
```

### 3. 覆盖率数据过大

```verilog
// 使用实例合并减少数据量
option.per_instance = 0;  // 合并所有实例
option.merge_instances = 1;
```
