# 复位类型与层次架构

> SoC 复位机制的分类体系与层次化设计，从全局复位到局部模块复位的完整图谱。

## 1. 复位基本概念

复位的根本目的：将半导体器件置于已知初始状态，避免随机启动不确定性。

### 关键技术方案

| 方案 | 原理 | 优缺点 |
|------|------|--------|
| 异步复位 | 直接连到触发器异步复位端口 | 响应快，但释放时可能亚稳态 |
| 同步复位 | 与时钟同步，在时钟边沿生效 | 无亚稳态，但时钟须稳定 |
| **异步复位同步释放** | 异步生效，同步释放 | **推荐方案**，兼顾响应与可靠性 |

### 与时钟、电源的交互

复位域通常与电源域层次结构对齐。电源域关闭/开启时，复位逻辑需正确处理避免状态损坏。

## 2. 全局复位类型

### 2.1 电源复位 (POR)

- **触发**：上电事件、PORn/RESETFULLn 引脚
- **范围**：全芯片，含测试/仿真逻辑
- **行为**：清除所有存储状态，锁存配置引脚
- 验证重点：复位时序、配置锁存、启动序列

### 2.2 硬复位 (Hard Reset)

- **触发**：引脚、RSCTRL/PLLCTL 寄存器、看门狗、仿真器
- **范围**：大部分功能逻辑，排除测试逻辑和复位隔离模块
- **行为**：重置寄存器/状态机，**保留内存内容**
- 验证重点：复位范围边界、内存完整性

### 2.3 软复位 (Soft Reset)

- **触发**：软件请求、特定寄存器写入
- **范围**：大部分功能逻辑，保留关键配置寄存器（PCIe MMRs、DDR EMIF MMRs）
- **行为**：重置核心与外设，保留系统配置
- 验证重点：保留寄存器完整性、软件可见一致性

### 2.4 冷复位 vs 温复位

| | 冷复位 | 温复位 |
|--|--------|--------|
| 深度 | 重置整个芯片 | 保留部分软件寄存器 |
| 启动过程 | 完整启动序列 | 跳过某些步骤 |

## 3. 局部复位类型

### 3.1 模块复位

- **范围**：单个功能模块
- **触发**：寄存器配置
- 需处理接口信号隔离
- 验证重点：复位后模块恢复、接口状态、不影响其他模块

### 3.2 局部复位

多核系统中允许单个核心独立重启而不影响其他核心。
验证重点：核心间通信一致性、共享资源访问控制、复位后重新同步。

## 4. 特殊复位类型

### 4.1 看门狗定时器复位

- 正常喂狗 → 不触发
- 异常超时 → 触发系统复位
- 验证重点：优先级处理、复位后看门狗重新初始化

### 4.2 错误监督复位

响应奇偶校验错误、ECC 错误、协议违规等。验证需构建错误注入场景。

## 5. 层次化复位架构

### 5.1 设计理念

以"始终开启域"为根，复位信号按功能和电源域分层组织：

```
系统级复位 ─┬─ 电源域复位1 ─┬─ 模块A复位
             │              └─ 模块B复位
             └─ 电源域复位2 ─┬─ 模块C复位
                            └─ 模块D复位
```

**优势**：支持局部复位、便于电源管理、简化复位网络、灵活配置。

### 5.2 系统级复位

最顶层全局复位，影响全芯片。
- 集成多个系统级复位源
- 处理优先级和仲裁
- 与电源管理单元协调时序

### 5.3 电源域级复位

- 每个独立电源域有独立复位控制
- 边界与电源域对齐
- 始终开启域的特殊处理：所有电源状态下复位逻辑正确

### 5.4 模块级复位

- 复位边界定义
- 接口信号处理（复位期间状态定义）
- 状态保持与恢复
- 复位依赖关系

### 5.5 层级交互规则

- 高优先级系统复位 → 传播到所有低层级复位域
- 低层级模块复位 → 不影响系统级复位状态
