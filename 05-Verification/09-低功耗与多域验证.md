# 低功耗与多域验证

> 时钟域（CD）、电源域（PD）、复位域（RD）三域交互已成为 SoC 验证的核心复杂度来源。

## 1. UPF 约束与时钟验证

### 1.1 UPF 标准

UPF (IEEE Std 1801) 通过 TCL 命令定义电源域、电源开关、隔离单元、保留单元、电平转换器等信息，确保低功耗设计意图正确实现。

### 1.2 与时钟验证的关联

- 低功耗设计（多电压域、DVFS）可能引入**新 CDC 路径**或破坏原有同步路径
- 跨电源域的时钟信号可能引入 CDC 问题
- 需要结合 UPF 信息进行 CDC 验证，支持实时仪器化和增量验证

## 2. 多域验证方法论

### 2.1 MDV 核心流程

```
UPF → 提取电源域信息
时钟树 → 提取时钟域信息
复位树 → 提取复位域信息
         ↓
验证域控制逻辑与交互正确性
```

### 2.2 三个验证层次

| 层次 | 内容 |
|------|------|
| 域结构验证 | 域划分是否正确 |
| 域控制验证 | 电源开关、时钟门控、复位控制逻辑 |
| 域交叉验证 | 跨域信号同步与隔离是否正确 |

### 2.3 MDV 优势

- RTL 级别早期发现域间问题
- 识别和验证跨时钟域的功率控制信号
- 全面验证所有域交互

## 3. 复位域交叉验证

### 3.1 RDC 问题本质

RDC 与 CDC 类似，但标准 CDC 工具通常无法发现。多异步复位信号场景下可能出现亚稳态和功能异常，即使在单时钟域内也可能发生。

### 3.2 验证方法

- **VC SpyGlass RDC**：静态时序与形式化混合验证
- **分级验证**：模块级 → SoC 级
- **流程**：读 RTL → 预编译 → 定义时钟/复位信号 → 检查设置 → 错误检查

### 3.3 关键关注点

- 复位撤离时序
- 未隔离单元
- 毛刺敏感电路
- PDC（电源域交叉）与 RDC 的交互

### 3.4 注意事项

- 早期介入，复杂度随设计成熟指数增长
- 需确保主时钟、复位、同步/异步性质的正确定义
- RDC 违规行为广泛且难识别，需系统化分析

## 4. 验证阶段策略

| 阶段 | 特点 | 时钟验证重点 |
|------|------|-------------|
| RTL 仿真 | 速度快，无延迟信息 | 时钟定义、复位、早期 CDC |
| 门级仿真 | 慢，有逻辑延迟 | 时序问题、异步时钟接口 |
| 形式验证 | 覆盖极端情况 | 时钟/复位连通性 |

## 5. 覆盖率与关闭标准

- CDC 覆盖率需基于故障模型和可观察性计算
- 分四个阶段，每阶段有明确目标和指标
- 业界高覆盖率阈值：80%+
- 关闭条件：无未解决 CDC 违规，所有同步路径已验证
