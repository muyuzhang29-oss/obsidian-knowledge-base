# 复位树与域交叉验证

> 复位树完整性检查、复位域交叉 (RDC) 验证、以及形式化/仿真/断言混合方法。

## 1. 复位树结构与挑战

复位树 = 复位信号从源到目标存储元件的网络拓扑。

### 1.1 基本组成

- 复位源节点（POR、看门狗、寄存器等）
- 复位分发节点（扇出点）
- 复位终端节点（触发器/锁存器复位端口）
- 复位同步器（跨时钟域同步）

### 1.2 关键挑战

| 挑战 | 说明 |
|------|------|
| 信号完整性 | 衰减、延迟、偏斜、毛刺、噪声耦合 |
| 异步分布 | 释放时序因偏斜不一致，可能违反 setup/hold |
| 复位域交叉 | 不同复位域间数据传输可能导致损坏 |

## 2. 复位树完整性验证

### 2.1 检查项

- **连通性**：每个存储元件是否连接到复位源
- **极性**：复位信号极性一致性
- **毛刺**：组合逻辑生成的复位毛刺
- **同步**：异步复位释放是否经过同步处理

### 2.2 静态验证

不依赖仿真激励，通过分析 RTL 代码结构验证：
- 复位网络无悬空/短路
- 未意外连接到非目标元件
- 信号宽度、多驱动检查

### 2.3 形式化验证

形式化验证比仿真更高效，能数学证明复位正确性。
- 证明复位传播正确性
- 验证释放后初始状态
- 检测死锁/活锁
- 检测 X 状态问题

## 3. 复位域交叉验证

### 3.1 RDC 风险

数据跨越不同复位域边界时，源域和目的域处于不同复位状态 → 数据损坏、亚稳态、协议违规、系统锁定。

### 3.2 RDC 处理技术

| 技术 | 说明 |
|------|------|
| 时钟门控 | 复位期间停止时钟 |
| 数据门控 | 使用使能信号控制传输 |
| 同步器 | 标准同步器处理跨域信号 |
| 复位协调 | 确保相关域同步复位 |

### 3.3 RDC 验证方法

- **静态分析**：识别所有 RDC 路径
- **形式化**：证明处理逻辑正确性
- **仿真**：构造 RDC 场景测试用例
- **架构分析**：评估策略合理性

## 4. 验证方法学

### 4.1 仿真验证

测试用例需覆盖：
- 基本功能：各类复位触发与响应
- 时序：建立/保持时间
- 并发：多复位源同时触发
- 恢复：释放后系统启动序列

**局限**：难以穷举，边界/竞争条件难触发。

### 4.2 形式化验证

**优势**：穷尽性、数学证明、早期发现深层问题。

应用场景：
- 复位连通性验证
- 复位行为验证（后状态、初始化序列）
- RDC 数据完整性
- 复位协议验证

### 4.3 断言验证

```systemverilog
property reset_state_check;
  @(posedge clk) reset_n |-> (state == IDLE);
endproperty

property reset_release_sequence;
  @(posedge clk) $fell(reset_n) |-> ##[1:10] (init_complete == 1);
endproperty
```

### 4.4 混合验证策略

| 阶段 | 方法 |
|------|------|
| 设计阶段 | 静态规则检查 |
| 模块验证 | 形式化验证 |
| 系统验证 | 仿真 |
| 回归 | 全部结合 |
