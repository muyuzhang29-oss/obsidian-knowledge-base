# 知识库健康检查脚本

> [!abstract] 概述
> 自动检测 Obsidian 知识库中的常见问题，生成 Markdown 格式的健康报告。

## 功能说明

| 检查项 | 说明 |
|--------|------|
| Frontmatter 检查 | 检测缺少 YAML Frontmatter 或 tags 字段的文件 |
| 断链检测 | 找出链接指向不存在笔记的 wiki 链接 |
| 孤立笔记 | 识别没有入链也没有出链的笔记 |
| 命名规范 | 检查文件名是否符合 `NN-名称.md` 规范 |
| 重复内容 | 检测内容完全相同或高度相似的文件 |

## 使用方法

### 基本运行

```powershell
cd D:\obsidian\knowledge-base
python 07-Scripts\03-知识库健康检查.py
```

### 自定义参数

```powershell
# 指定知识库目录
python 07-Scripts\03-知识库健康检查.py --kb-dir "D:\other\vault"

# 指定报告输出路径
python 07-Scripts\03-知识库健康检查.py --output "D:\report.md"
```

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--kb-dir` | `D:\obsidian\knowledge-base` | 知识库根目录 |
| `--output` | `07-Scripts/健康检查报告.md` | 报告输出路径 |

## 报告输出

脚本运行后会在 `07-Scripts/健康检查报告.md` 生成报告，包含：

1. **健康评分** - 满分 100 分，每个问题扣 2 分
2. **问题汇总** - 各检查项的问题数量和状态
3. **详细列表** - 每个问题的文件路径和具体描述
4. **改进建议** - 针对发现问题的修复建议

## 技术实现

- 使用 `pathlib` 处理文件路径
- 使用正则表达式匹配 Frontmatter 和 wiki 链接
- 基于内容哈希检测重复文件
- 自动跳过 `.obsidian`、`.claude` 等隐藏目录
- 仅依赖 Python 标准库，无需安装额外包

## 注意事项

> [!warning] 运行要求
> - 需要 Python 3.8+
> - 报告文件会被覆盖，重要报告请备份
